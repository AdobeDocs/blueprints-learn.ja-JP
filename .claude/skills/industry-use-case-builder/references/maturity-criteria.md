# ユースケースの成熟度評価条件

## 成熟度レベル

### 基本

**バッジ：** `[!BADGE Foundational]{type=Neutral}`

次のような場合のユースケース **基礎知識** です。

- 単一ステップまたはイベントトリガーパターン（イベントトリガーメッセージ、バッチ送信メッセージのアクティベーション）にマッピングされます
- わかりやすいデータ要件（単一イベントタイプ、標準プロファイル属性）がある
- AI/ML の機能が最小限である、または必要ない
- 複雑なオーケストレーションを行わずに、標準のチャネル配信（メール、SMS、プッシュ）を使用する
- 明確なベストプラクティスを伴う確立された実装パターンがある
- 必要なシステム統合が少ない（通常は 1～2 個のデータソース）

**一般的なパターン：** イベントトリガーメッセージ、バッチ送信メッセージのアクティブ化

**例：** 放棄された買い物かごの回収、予定のリマインダー、サービス通知、価格低下アラート、請求の支払いリマインダー

### 新興

**バッジ：** `[!BADGE Emerging]{type=Informative}`

次のような場合にユースケースが **新たに出現** します。

- 複数ステップまたはパーソナライゼーションパターン（複数ステップのオーケストレーションされたジャーニー、既知の訪問者のPersonalization、行動レコメンデーション、匿名訪問者のPersonalization）にマッピングします
- 行動データの収集と分析が必要
- レコメンデーションモデルまたは既知の訪問者プロファイル解像度を使用します
- 分岐ロジックを持つマルチタッチ育成シーケンスが含まれます
- 統合が適度に複雑（データソース数 2～4）
- オーディエンスアクティベーションまたはセグメントベースのターゲティングが必要になる場合があります。

**典型的なパターン：** 複数ステップのオーケストレーションされたジャーニー、既知の訪問者 Web/アプリPersonalization、行動レコメンデーション、匿名訪問者 Web Personalization、B2B Audience Activation

**例：** ようこそシリーズ、服薬遵守キャンペーン、パーソナライズされたアカウントダッシュボード、リードスコアリング、コンテンツレコメンデーション

### 詳細

**バッジ：** `[!BADGE Advanced]{type=Caution}`

次のような場合のユースケースは **詳細** です。

- 意思決定またはクロスチャネルオーケストレーションパターン（Decisioning を使用したクロスチャネルジャーニー、Offer Decisioning）にマッピングします
- AI を利用した予測または傾向スコアリングが必要
- 複数のチャネルにわたって一元的な決定ロジックを同時に使用
- 複数のジャーニーポイントでのリアルタイム決定を使用した複雑なオーケストレーションが必要
- 統合が複雑である（4 つ以上のデータソース、リアルタイムデータフィード）
- 高度なビジネス・ルール、優先度管理、頻度ガバナンスが必要

**典型的なパターン：** Decisioning を使用したクロスチャネルジャーニー、Offer Decisioning

**例：AI スコア付** によるチャーン防止、ライフステージの製品レコメンデーション、クロスセル最適化、ロイヤルティプログラムのパーソナライゼーション、VIP限定オファー

## 評価ワークフロー

1. ユースケースがマッピングされるユースケースパターンを特定する
2. クイック一致については、上記の「典型的なパターン」のリストを確認してください
3. パターンが成熟度を明確に示さない場合は、データの複雑さ、AI/ML 要件および統合の範囲を評価します
4. 「パターンマッピング（{pattern}）と {key factors} に基づいて、{reasoning} の理由で {level} と分類します。」という理由を付けて、評価をユーザーに提示します。
5. 続行する前に、ユーザーに確認または上書きしてもらってください
