# Adobe Experience League オーサリングガイドライン

Source:https://experienceleague.adobe.com/en/docs/authoring-guide/using/home
クロール:2026-03-15

---

## &#x200B;1. メタデータ/前面

### 必須フィールド
| フィールド | 要件 | ルール |
|---|---|---|
| `title` | 必須 | 最大 60 文字（英語）。 タイトルケース。 わかりやすくするために必要でない限り、製品名は付けません。 システムが「」を追加 | ProductName&quot;は自動的に削除されます。 コロンまたは角括弧が含まれている場合は引用符で囲みます。 |
| `description` | 必須 | 最大 150～160 文字。 空白または null にすることはできません。 概念は、「詳細」で始まります。 タスクの開始「学習方法」 または命令動詞。 |

### オプションの、よく使用されるフィールド
| フィールド | 有効な値 | 備考 |
|---|---|---|
| `feature` | 機能の YAML 値と一致する必要があります。タイトルケースとスペースは一致します（例：「コンテンツフラグメント」） | 記事ごとに 1～2、トピックで表示 |
| `feature-set` | 機能 YAML に対して検証する必要があります | 機能の親アプリケーション |
| `role` | ユーザー，開発者，リーダー，管理者，アーキテクト，データアーキテクト，データエンジニア | 大文字と小文字を区別。「作成対象」と表示される |
| `level` | 初心者、中級、経験者 | 指定しない場合、デフォルトは Beginner になります |
| `solution` | ソリューション YAML に一致する必要があります（例：「Campaign、Campaign Standard v7」） | 複数の値を使用できます。検索フィルタリングに使用されます |
| `topic` | トピックの YAML と一致する必要があります。タイトルの大文字と小文字をスペースで指定します | クロス製品フィルタリングの場合（「統合」など） |
| `type` | ドキュメント、チュートリアル、トラブルシューティング | リポジトリレベル。デフォルトはドキュメント |
| `short-description` | 1 文、最大 20 語 | 説明が長すぎる ExL ランディングページの場合 |
| `badgePremium` | `label="Premium" type="Positive" url="..."` | ページレベルのバッジ |
| `mini-toc-levels` | 1 ～ 6 （デフォルト：2） | 右ナビゲーション見出しの表示深度をコントロールします |
| `hidefromtoc` | true/false | 左ナビゲーションから除外しますが、URL はアクセス可能なままです |

### メタデータの配置
- 記事レベル：Markdown ファイルの先頭（YAML の最前線）
- 目次レベル：目次.md ファイル内
- リポジトリレベル：metadata.md ファイル内

### 主要な書式設定ルール
- コンマまたは角括弧を含んだ値を引用符で囲む
- コンマで区切られた複数の値
- YAML 定義に対して大文字と小文字を区別した照合
- タグ値が空または空の場合、検証が失敗します

### 非推奨フィールド
seo-title、seo-description、audience、difference、uuid （移行時代から）

## &#x200B;2. MARKDOWN 構文（ADOBE固有）

### テキストフォーマット
- 太字：`**text**`
- 斜体：`*text*`
- 太字+斜体：`***text***`
- エスケープ特殊文字：`# { } [ ] * + - . !` の前にバックスラッシュ `\` を使用します
- HTML エンティティ：`&lt;` `&gt;` `&amp;` `&mdash;` `&ndash;`

### 見出し
- ATX スタイルを使用する（`#` 構文のみ、アンダーライン/セテキストなし）
- ドキュメントごとに 1 つの H1 （通常、ページタイトルはメタデータ `title` と一致）
- 後続のすべての見出し H2 （`##`）以下
- 見出しの前後に必要な空白行（最初の見出しを除く）
- 最大 69 文字（英語）/120 文字（ローカライズ）
- 最大 5 単語を優先します
- すべての見出し用の文の大文字/小文字（最初の単語と固有名詞のみ大文字）
- `title` メタデータフィールドのみのタイトルケース
- 終わりの句読点なし（FAQ の見出しには疑問符を使用できます）
- Custom anchor IDs: `## Title {#custom-id}` – ピリオドではなくハイフンを使用します。
- ドキュメントに重複する見出しアンカー ID がありません
- 予約されたアンカー名を避ける：検索、結果、ファセット、ページネーション、並べ替え、クエリ、searchbox、コンテンツ、ヘッダー、フッター、メイン、ナビゲーション、サイドバー、ページ、コンテナ、ラッパー
- すべての見出しに対して、本文の段落を 1 つ以上続けて入力します（見出しの直後にリストやテーブルを配置しないでください）
- 概念の見出し：名詞/名詞句
- タスクの見出し：必須動詞（動名詞ではなく – 「セグメントを作成」ではなく「セグメントを作成」）
- 見出しレベル間の並列構造

### リスト
- 箇条書き（順不同）:`*`、`-`、`+` – 文書内で一貫して使用します。
- 順序付き：各品目の `1.` （自動番号）
- リストの前後の空白行
- 入れ子になった番号付き項目を 3 スペース、入れ子になった箇条書きを 2 スペースでインデントします。
- 箇条書きの句読点：末尾のセミコロン/コンマ/結合を省略します。完全な文にのみピリオドを追加します
- すべての項目が≤3 語であるか、UI ラベル/見出しである場合は、ピリオドを追加しないでください

### リンク
- 外部：`[text](https://url.com)`
- 相対（同じレベル）: `[text](file.md)`
- ルート相対：`[text](/help/path/file.md)` — `/` で始める必要があります
- ディープリンク（同じページ）:`[text](#heading-anchor)`
- クロスドキュメントディープリンク：`[text](file.md#heading-id)`
- 新しいタブを強制：`[text](url){target="_blank"}`
- ベア URL：山かっこ `<https://example.com>` で囲んでクリック可能にする
- 参照リンク：参照スタイルリンクでは絶対リンクのみが機能します
- 相対リンクを介して複数の目次の場所に同じファイルを含めない
- Windows ユーザー：バックスラッシュをパスのスラッシュに変換する

### 画像
- 基本：`![alt text](image.png "hover tooltip")`
- サイズ変更：`{width="300"}`
- 位置合わせ：`{align="center"}` または `{align="right"}`
- ズーム可能：`{zoomable="yes"}` または `{modal="regular"}`
- 最大ファイルサイズ：100 MB （5 MB 未満をお勧めします）
- 代替テキストは、すべての画像（SEO およびアクセシビリティ向け）に必須です
- 画像ファイル名：小文字をハイフンに変換
- 指定されたアセットフォルダーへの画像の保存

### コードブロック
- Inline: バッククォート `` `code` ``
- ファイル名、URL、ユーザー定義用語、コマンドに使用
- 囲みコードブロック：言語識別子を持つ 3 つのバッククォート

  ```
  ```javascript
  code here
  ```

  ```
  
  
- オプション：`{line-numbers="true"}`、`{start-line="7"}`、`{highlight="11-13, 16"}`
- 囲みコードブロックには常に言語識別子を含める

### テーブル
- 区切り行には、列ごとに少なくとも 3 つのハイフンが必要です：`|---|---|---|`
- 配置：左 `|---|`、中央 `|:---:|`、右 `|---:|`
- リテラルパイプをエスケープ：`&vert;` を `\|` 用または使用
- HTML テーブルが許可されています。`<table style="table-layout:auto">` または `table-layout:fixed` を使用してください
- HTML テーブルの整列：`<td align="left|center|right">`
- HTML テーブル内に Markdown 構文を使用しないでください（例：`[!NOTE]` はHTML テーブルでは機能しません）
- 罫線の削除：`<tr style="border: 0;">`
- マークダウン表のレイアウトオプション：空白行を含む表の後に `{style="table-layout:auto"}` を追加
- 水平スクロールバーの表示の問題により、非常に広いテーブルや高いテーブルを使用しない

## &#x200B;3. 特別なADOBE構文拡張

### コールアウト/勧告

```
>[!NOTE]
>
>Text here.

>[!TIP]
>
>Text here.

>[!IMPORTANT]
>
>Text here.

>[!WARNING]
>
>Text here.

>[!CAUTION]
>
>Text here.

>[!ADMIN]
>[!AVAILABILITY]
>[!PREREQUISITES]
>[!INFO]
>[!ERROR]
>[!SUCCESS]
```
- 重大：`>` と `[!` の間にスペースがありません – `>[!NOTE]` NOT `> [!NOTE]` を使用してください
- `>[!NOTE]` と本文テキスト行の間に空白行を追加

### タブ

```
>[!BEGINTABS]

>[!TAB iOS]
Content here

>[!TAB Android]
Content here

>[!ENDTABS]
```

### 折りたたみ可能なセクション（アコーディオン）

```
+++Click to expand
Content inside
+++
```
メモ：ネストされた折りたたみ可能なセクションはサポートされていません。

### シェードボックス

```
>[!BEGINSHADEBOX "Optional Title"]
Content here
>[!ENDSHADEBOX]
```

### ビデオ

```
>[!VIDEO](https://video.tv.adobe.com/v/ID/?quality=12&learn=on)
```
トランスクリプトの `{transcript=true}` を追加します。

### その他の関連リソース

```
>[!MORELIKETHIS]
>* [Article 1](url)
>* [Article 2](url)
```

### コンテキストヘルプ

```
>[!CONTEXTUALHELP]
>id="..."
>title="..."
>abstract="..."
>additional-url="..."
```

### バッジ （インライン）

```
[!BADGE Label]{type=Informative url="https://example.com" tooltip="text"}
```
タイプ：`Informative` （青）、`Positive` （緑）、`Negative` （赤）、`Neutral` （グレー）、`Caution` （黄）

### テキストのハイライト表示（プレビュー）

```html
<span class="preview">highlighted text</span>
```

### およびのスニペットを含む
- ファイル全体を含める：`{{$include /help/_includes/legal-blurb.md}}`
- スニペット （見出しなし）:`{{snippet-id}}`
- 再利用可能なファイル `help > _includes/` フォルダーに保存
- `help > _includes/snippets.md` に保存されているスニペット
- Includes は見出しを持つことができます；スニペットは持つことができません
- 同じリポジトリ内でのみ機能します（クロスリポジトリでは機能しません）
- 非参照ファイルでの `{{` または `}}` 文字のエスケープ

### DNL （Do Not Localize）タグ
- `[!DNL ProductName]` – 翻訳しない製品名またはブランド名を折り返します。
- ページ内の最初のメンションまたは目立つメンションでの使用

### UICONTROL タグ
- `[!UICONTROL Button Label]` - UI 要素のテキスト （ボタン、メニュー、フィールド名）を折り返します。
- 機械翻訳ではなく製品データベースから UI 文字列を確実に取り込みます
- UICONTROL タグ内にアポストロフィ、引用符、句読点がない
- 手順で参照される UI 要素に太字書式を使用します

### サポートされていない機能
- タスクリスト（チェックボックス）
- 絵文字
- 罫線（横）
- ネストされた折りたたみ可能なセクション

## &#x200B;4. ファイル名とフォルダー構造

### ファイル命名規則
- 小文字のファイル名（ハイフンのみ）
- 大文字、アンダースコア、ピリオド、スペースを使用しない
- わかりやすい SEO わかりやすい名前（例：`cmo.md` で `calculated-metric-overview.md` ない）
- ファイル名には数字を使用せず、わかりやすい単語を使用する
- 予約済みまたは競合する名前を使用しない：`metadata.md`、`search.md`
- ライブファイル名を変更するには、リダイレクト管理が必要です（URL の変更）

### フォルダー構造
- ディレクトリ/フォルダー名は URL に影響しません（リポジトリ名とファイル名のみが URL に影響します）
- TOC.md は、Git フォルダーの場所ではなく、ナビゲーション階層を制御します
- 指定されたアセットフォルダーへの画像/アセットの保存
- `help/_includes/` に保存された再利用可能なインクルード
- `help/_includes/snippets.md` に保存されているスニペット

### TOC.md ルール
- Experience Leagueでレンダリングするには、すべての記事を TOC.md にリストする必要があります。
- H1 形式：`# Guide Title {#anchor-id}` — アンカーは、ベース URL パスを生成します。
- セクション見出しにはアンカー ID が必要：`+ Section Name {#section-id}`
- セクション見出し（親項目）をリンク自体にすることはできません
- 記事のリンク：`+ [Title](path/file.md)`
- セクションアンカー ID を変更すると、ネストされたすべての記事の URL が変更されます（リダイレクトが必要です）
- 全体（`+`、`*`、`-`）で一貫した箇条書きスタイルを使用する
- 目次メタデータ：`user-guide-description` （オプションで `breadcrumb-title`）
- `mini-toc-levels`：右ナビゲーションの見出しの表示を制御します（1～6、デフォルトは 2）。

## &#x200B;5. コンテンツ品質および編集基準

### 音声とトーン
- 受動態より能動態の方が望ましい
- 現在時制を未来時制より優先する
- 教育用コンテンツの場合は「あなた」
- 35 語未満の文
- 強力で正確な動詞 – 弱い副詞（「非常に」、「極端に」）を避ける

### 手順/手順
- 各手順は、簡潔な 1 つのコマンド（1 文）です。
- 追加情報は次の行に移動します（手順でインデント）
- 各手順を命令動詞から開始する
- 手順を約 7 ステップに制限します（サブタスクを分割する前に最大 10 ステップまで）
- 手順の前に概念情報を配置する
- 関連する手順の後にスクリーンショットを配置
- 読者の向きのためにページ名やタブ名を繰り返す
- 手順の UI 要素に対する太字の UICONTROL 書式設定

### インターフェイス用語
- **開く/閉じる**：アプリケーションとプログラム
- **移動**：メニュー、タブ、UI の場所
- **選択**：リストのテキスト、セルまたはオプション
- **クリック**：マウスアクション（必要でない限り、コントロールタイプを指定しないでください）
- **ドロップダウンメニュー** （「ドロップダウンリスト」ではない）

### 製品名
- わかりやすくするために必要でない限り、タイトルや見出しに製品名を追加しないでください
- 製品名の前の「the」を省略
- ガイドレベルで最初に言及された「Adobe」を含める（商標要件）
- 法的に必要でない限り、セカンダリメンションに「Adobe」をドロップ
- 製品名に DNL タグを使用して誤翻訳を防ぐ

### 強調と書式設定
- 太字：UI 要素とキーボードアクション
- 斜体：エラーメッセージ、外来語、強調
- コード（バッククォート）：ファイル名、URL、ユーザー定義用語
- UI 要素を囲む引用符はありません（代わりに UICONTROL タグを使用）

### 大文字
- 見出し、ナビゲーション、サブ見出しの文の大文字と小文字
- メタデータフィールドのタイトルケ `title` スのみ
- 固有名詞は常に大文字

## &#x200B;6. SEO のベストプラクティス

- ページごとに 1 つの H1 – 簡潔で説明的であり、ページの概要をユーザーに伝える必要があります。
- 順次見出し階層（h1→h2→h3、スキップレベルなし）
- H1、本文、メタデータ（`keywords` メタタグではない）にキーワードを含める（Googleでは無視されます）
- キーワードスタッフィングを避ける（インテント/頻度）
- タイトル：最大 50～60 文字、システムによって「| Adobe ProductName」が自動的に追加される
- 説明：150～160 文字。call to actionにする必要があります。コンテンツの繰り返しではありません
- すべての画像に説明的な代替テキストを追加
- ビデオトランスクリプトを含める（検索エンジンはテキストの読み取りのみ可能）
- 関連ページへの戦略的なリンク（孤立したページを避ける）
- 構造化要素（番号付きリスト、サブ見出し、コードブロック）を含める
- AnswerThePublic やGoogle Trends などのツールを使用して、キーワードを調査します
- コンテンツには、E-E-A-T （経験、専門知識、権威、信頼性）を示す必要がある

## &#x200B;7. 場所

### 最初に機械翻訳
- 英語のソースコンテンツは、ローカライズされたバージョンを自動生成します
- サポートされる言語：ドイツ語、フランス語、日本語、イタリア語、スペイン語、ポルトガル語（ブラジル）、簡体字中国語、繁体字中国語、韓国語、オランダ語、スウェーデン語

### ローカリゼーション用の書き込み
- 一貫した用語
- 短い文と活発な声
- 標準の英語の語順（主語→動詞→オブジェクト）
- 相対代名詞（&quot;that&quot;、&quot;which&quot;）を使用します
- 避ける：ユーモア、イディオム、専門用語、地域用語、隠喩、不要な言葉

### ローカリゼーションタグ
- `[!UICONTROL Label]` - UI 文字列が機械翻訳ではなく、製品 DB から取り込まれるようにします。
- `[!DNL ProductName]` – 製品/ブランド名が翻訳されないようにします。
- 「do-not-localize」フォルダー内の画像は、ローカライゼーションから除外されます

## &#x200B;8. コンテンツタイプ

- **ガイド**:TOC で参照されるページのオンラインコレクション（公式のベストプラクティスを説明）
- **チュートリアル**：特定のユースケースに関する教育コンテンツ（ビデオまたはテキスト）。ラーニングリポジトリで公開
- **リファレンスガイド**：開発者 API/SDK。通常はテーブル形式。developer.adobe.comに公開されます
- **コース**：特定のユーザーの役割を対象とした、エキスパートによるレッスンのコレクション
- **ナレッジベース記事**：一時的に関連する簡単なトラブルシューティングコンテンツ
- **ランディングページ/ホームページ**：別管理（SCCM）

## &#x200B;9. 回避すべき一般的な検証エラー

- `title` または `description` のメタデータが見つからない、または空白である
- 「詳細情報」で始まら `description` い または「方法を学ぶ」を参照してください。
- コールアウト構文の `>` と `[!` の間のスペース （`>[!NOTE]` ではなく `> [!NOTE]`）
- 太字のスペース マーカー：`**text **` （末尾のスペースは太字で区切ります）
- HTML テーブル内の Markdown 構文（例：コールアウトが機能しない）
- ドキュメント内の見出しアンカー ID の複製
- ピリオドを含むアンカー ID （代わりにハイフンを使用）
- 予約されたアンカー名（検索、結果、ヘッダー、フッターなど）の使用
- YAML 定義に一致しない `role`、`level`、`feature`、`topic` 値
