# B2B オーディエンスのアクティベーション

このガイドでは、[!DNL Adobe Real-Time Customer Data Platform] （[!DNL RT-CDP]）B2B editionを使用してアカウントベースのB2B オーディエンスをアクティブ化するための包括的な実装リファレンスを提供します。 web、電子メール、広告、CRM チャネルをまたいで、アカウントレベルのオーディエンスを構築、評価、活性化する必要があるソリューションアーキテクト、マーケティングテクノロジスト、実装エンジニア向けに設計されています。

アカウントプロファイルの統合から、オーディエンスの評価とアクティベーション、そして[!DNL Marketo Engage]、[!DNL LinkedIn]、CRM システムなどのB2B固有の宛先に至るまで、ライフサイクル全体をカバーしています。 実行可能なあらゆる導入アプローチには、トレードオフと意思決定ガイダンスが提示されており、自社に適したパスを選択するのに役立ちます。

## ユースケースの概要

B2B マーケティング部門は、個人レベルではなく、アカウントレベルでオーディエンスをターゲティングし、活性化する必要があります。 ターゲティングの単位が単一の消費者プロファイルであるB2C オーディエンスのアクティベーションとは異なり、B2B オーディエンスのアクティベーションには、人物と所属するアカウントの関係を把握し、アカウントレベルの属性と人物レベルのエンゲージメントシグナルを組み合わせてオーディエンスメンバーシップを評価し、アカウントベースのターゲティングをサポートする配信先にオーディエンスを配信する必要があります。

[!DNL RT-CDP] B2B editionは、アカウント、オポチュニティ、キャンペーンに特化したXDM クラスと、個人とアカウントの関係をマッピングするB2B ID解決を含む標準[!DNL Real-Time Customer Data Platform]を拡張します。 これによりマーケターは、アカウントに関連する人々の企業特性データ（業界、収益、従業員数）、技術特性データ（テクノロジースタック、製品の使用状況）、行動データ（web訪問、メールエンゲージメント、イベントへの参加）を組み合わせたアカウントオーディエンスを構築することができます。

アクティブ化されたアカウントオーディエンスは、需要創出funnelのユースケースを強化します。[!DNL LinkedIn]のfunnel上部の認知キャンペーン、ディスプレイ広告、[!DNL Marketo Engage]のfunnel中央のナーチャリングプログラム、CRM統合によるfunnel下部のセールスイネーブルメントです。 アカウント抑制オーディエンスは、既存顧客、クローズドまたはアクティブな販売サイクルにあるアカウントを除外することで、支出の無駄を防ぎます。

>[!NOTE]
>ユースケースで、アカウントレベルではなく個人レベル（B2C）でオーディエンスをアクティブ化する場合は、[宛先へのオーディエンスアクティベーション &#x200B;](audience-activation-to-destinations.md)を参照してください。 このパターンでは、標準のRT-CDP データモデルが使用され、B2B editionは必要ありません。

## 主なビジネス目標

このユースケースパターンでは、次のビジネス目標をサポートしています。

### リードジェネレーションの促進

フォーム、イベント、コンテンツ、マルチチャネルエンゲージメントを通じて、セールスパイプラインの質の高いリードをより多く生み出します。

**KPI:**&#x200B;見込み顧客、リード単価、リードコンバージョン

[リードジェネレーションの強化について詳しく見る](/help/blueprints/business-objectives/acquisition-growth/increase-lead-generation.md)

### リードのクオリフィケーションとコンバージョンを向上

スコアリング、育成、パーソナライズされたフォローアップにより、リードの品質を向上させ、パイプラインの進行を加速させることができます。

**KPI:** リードのコンバージョン、見込み顧客/リードのコンバージョン、効率性

[リードのクオリフィケーションとコンバージョンの改善について詳しく見る](/help/blueprints/business-objectives/qualification-sales-b2b/improve-lead-qualification-conversion.md)

### 新規顧客の獲得

ターゲットを絞った獲得キャンペーン、類似オーディエンス、有料メディアの最適化により、顧客基盤を拡大できます。

**KPI:**&#x200B;新規顧客、顧客獲得コスト、見込み顧客/リードコンバージョン

[新規顧客の獲得についてさらに詳しく](/help/blueprints/business-objectives/acquisition-growth/acquire-new-customers.md)

### マーケティングの支出とROIの最適化

ターゲティング、アトリビューション、オーディエンスの抑制、予算配分の改善を通じて、マーケティングのROIを向上できます。

**KPI:** コスト削減、顧客獲得コスト、増分収益

[マーケティング費用とROIの最適化について詳しく見る](/help/blueprints/business-objectives/cost-efficiency/optimize-marketing-spend-roi.md)

## 戦術的なユースケース

次のシナリオは、このパターンを実際にどのように適用できるかを示しています。

- **アカウントベースの広告[!DNL LinkedIn]** — [!DNL RT-CDP] B2B editionからアクティブ化されたアカウントリストを使用して、理想的な顧客プロファイル（ICP）とスポンサーコンテンツおよび[!DNL LinkedIn]のInMail キャンペーンを一致させるターゲットアカウント
- **[!DNL Marketo Engage]ナーチャリングプログラムのターゲティング** — アカウントレベルのクオリフィケーション基準に基づいて、関連するリードと連絡先をターゲットナーチャリングストリームに登録するために、[!DNL Marketo Engage]にアカウントオーディエンスをアクティブ化します
- **CRM アカウントリストの同期** – 営業チームの可視化、担当地域の割り当て、アウトバウンドの見込み顧客ワークフローのために、適格アカウントリストを[!DNL Salesforce]または[!DNL Microsoft Dynamics]にプッシュします
- **有料メディアのアカウント抑制** – 有料獲得キャンペーンから既存の顧客、契約成立したアカウント、またはアクティブな販売サイクルのアカウントを抑制して、無駄な支出を削減します
- **インテントベースのアカウントターゲティング** — サードパーティのインテントシグナルとアカウントレベルでのファーストパーティのエンゲージメントデータを組み合わせて、市場内アカウントのオーディエンスを特定し、アクティブ化します
- **製品の既存アカウントへのクロスセル** – ある製品ラインを使用してアカウントのオーディエンスを構築するが、別の製品ラインを使用しない場合は、クロスセルキャンペーン用にメールおよび広告チャネルに対してアクティブ化する
- **イベントおよびウェビナーのターゲティング** – 広告およびメールチャネルにアカウントオーディエンスをアクティベートして、ターゲットアカウントからのイベント登録を促進します
- **競合他社への乗り換えキャンペーン** – 広告およびメールチャネルを通じて有効化された、カスタマイズされたメッセージを含む競合他社製品を使用してアカウントをターゲットにする
- **階層化されたアカウントエンゲージメント** – 集約された個人レベルのアクティビティに基づいてアカウントをエンゲージメント層（高、中、低）にセグメント化し、各層で差別化されたキャンペーンをアクティブ化します
- **パートナー共同マーケティングオーディエンス** — アカウントオーディエンスセグメントをチャネルパートナーまたはクラウドストレージの宛先を通じた共同マーケティングプログラムと共有します

## ユースケースパターン

**B2B オーディエンスのアクティブ化**

web、電子メール、広告のチャネルをまたいで、アカウントベースのB2B オーディエンスを活用できます。

**関数チェーン：** Account Profile Enrichment > Account Audience Evaluation > Destination Configuration > Audience Activation > Monitoring

## アプリケーション

このユースケースパターンを実装するには、次のアプリケーションを使用します。

- **[!DNL Real-Time CDP]B2B edition** — アカウントプロファイルの統合、B2B ID解決、アカウントオーディエンスの評価、B2B固有の宛先設定、アカウントオーディエンスのアクティベーションのためのコアプラットフォーム
- **[!DNL Adobe Experience Platform]（AEP）** — B2B XDM データモデリング、CRMおよびマーケティングオートメーションソースからのデータ取り込み、ID サービス、ガバナンスのための基盤インフラストラクチャ
- **[!DNL Marketo Engage]** – アクティブなアカウントオーディエンスによって提供される、リードナーチャリングプログラム、スコアリング、キャンペーン実行のプライマリB2B マーケティングオートメーションの宛先

## 実装オプション

次のオプションでは、このユースケースパターンを実装するための様々なアプローチについて説明します。 各オプションを確認し、要件に最適なものを選択します。

### オプション A: オーディエンス アクティベーションを[!DNL Marketo Engage]にストリーミングする

**B2B マーケティングオートメーションプラットフォームとして[!DNL Marketo Engage]を利用している**&#x200B;企業に最適です。トリガーナーチャリングプログラムをほぼリアルタイムでオーディエンスメンバーシップを更新したり、リードスコアを更新したり、アカウントの適格性や適格性が低い場合にキャンペーンメンバーシップを変更したりすることが必要です。

**仕組み：**

このオプションは、[!DNL RT-CDP]のネイティブ [!DNL Marketo Engage]宛先コネクタを使用して、アカウントオーディエンスメンバーシップの変更を[!DNL Marketo Engage]に直接ストリーミングします。 アカウントがオーディエンスセグメントの対象となるか、離脱すると、[!DNL Marketo]の関連するリードと取引先責任者がセグメントメンバーシップ属性で更新されます。[!DNL Marketo] スマートキャンペーンは、これらのメンバーシップの変更にもとづいてトリガーすることができます。

[!DNL Marketo Engage]宛先はストリーミング宛先です。つまり、オーディエンスメンバーシップの変更は、スケジュールされたバッチではなく、発生するにつれて段階的に送信されます。 これにより、アカウントの選定の変更に対応する必要があるキャンペーンを開始するまでの時間を短縮できます。 フィールドマッピングは、[!DNL RT-CDP]個のプロファイル属性を[!DNL Marketo]個のリード/連絡先フィールドに接続し、[!DNL RT-CDP]からのアカウントレベルのデータを使用して[!DNL Marketo]件のレコードのエンリッチメントを可能にします。

**重要な考慮事項：**

- API アクセスを持つアクティブな[!DNL Marketo Engage] サブスクリプションが必要です
- ストリーミングアクティベーションでは、オーディエンスの完全なスナップショットではなく、段階的な更新を送信する
- [!DNL Marketo]の個人レベルのレコードは、アカウントレベルのレコードではなく、直接更新されます
- [!DNL Marketo]件のスマートキャンペーンは、セグメントメンバーシップ フィールドの更新に対応するように設定する必要があります

**利点：**

- [!DNL Marketo]でほぼリアルタイムのオーディエンスメンバーシップの更新
- ネイティブ双方向コネクタで統合を簡素化
- 増分更新によるデータ転送ボリュームの最小化
- [!DNL Marketo]さんは、オーディエンスの変化に基づいてナーチャリングプログラムを即座にトリガーできます

**制限：**

- アクティブ化ターゲットとして[!DNL Marketo Engage]に制限
- ストリーミング評価の適格性の制約は、基礎となるオーディエンス定義に適用されます
- [!DNL RT-CDP]個のアカウントオーディエンスと[!DNL Marketo]個のリード/連絡先レコードのマッピングが必要です
- 個人レベルの更新をアカウントレベルのアクションに変換するには、複雑な[!DNL Marketo] プログラムロジックが必要になる場合があります

**Experience League:**

- [Marketo Engageの宛先](https://experienceleague.adobe.com/en/docs/experience-platform/destinations/catalog/adobe/marketo-engage)
- [Marketo Engageの宛先に対してオーディエンスをアクティブ化](https://experienceleague.adobe.com/en/docs/experience-platform/destinations/catalog/adobe/marketo-engage#activate)

### オプション B：広告プラットフォームへのオーディエンスのバッチアクティベーション

**毎日のオーディエンスの更新で十分であり、リアルタイムの応答ではなく、アカウントレベルのリーチと認知度が主な目標となる、[!DNL LinkedIn]やディスプレイネットワークなどの広告プラットフォームでのアカウントベースの広告キャンペーンに最適です。**

**仕組み：**

このオプションは、スケジュールされたバッチケイデンスで、広告プラットフォームの宛先（[!DNL LinkedIn]一致オーディエンス、[!DNL Google]顧客マッチ、[!DNL Facebook] カスタムオーディエンス、または[!DNL The Trade Desk]）にアカウントオーディエンスをアクティブ化します。 アクティベーションデータフローは、アカウントオーディエンスメンバーを、広告プラットフォームがユーザーベースとの照合に使用する、マッピングされたID フィールド（通常は関連する連絡先のハッシュ化されたメールアドレス）で書き出します。

特に[!DNL LinkedIn]の場合、[!DNL LinkedIn]一致オーディエンスの宛先は、電子メールベースのマッチングに加えて、企業名とドメインベースのマッチングを受け入れ、真のアカウントレベルのターゲティングを可能にします。 その他の広告プラットフォームでは、通常、対象アカウントに関連する連絡先から個人レベルのID （電子メール、電話）を必要とします。

バッチアクティベーションは、設定可能なスケジュール（毎日、6時間ごと）で実行されます。 宛先のタイプに応じて、完全なオーディエンススナップショットまたは増分変更を書き出すことができます。 このアプローチは、オーディエンスの鮮度の要件を数分ではなく数時間で測定する、継続的な認知度向上キャンペーンに適しています。

**重要な考慮事項：**

- オーディエンスの鮮度は、バッチスケジュール（通常は毎日）によって異なります
- Advertisingでは、独自の一致率の制限があります
- [!DNL LinkedIn]はアカウントレベルの照合をサポートしています。他のプラットフォームでは個人レベルのIDが必要です
- 書き出しファイル形式とID要件は宛先によって異なります

**利点：**

- 複数の広告プラットフォームを同時にサポート
- バッチ処理で大量のオーディエンスを効率的に処理
- アカウント抑制オーディエンスで広告費の無駄を削減
- [!DNL LinkedIn]件のマッチドオーディエンスにより、ネイティブアカウントレベルのターゲティングが可能になります

**制限：**

- オーディエンスの更新はリアルタイムではなく、待ち時間はバッチスケジュールに依存します
- 一致率はプラットフォームや利用可能なID フィールドによって異なります
- 一部のプラットフォームは、真のアカウントレベルのマッチングをサポートしておらず、個人レベルのマッチングのみをサポートしています
- 広告プラットフォームごとに個別の宛先設定が必要

- [Google カスタマーマッチの宛先](https://experienceleague.adobe.com/en/docs/experience-platform/destinations/catalog/advertising/google-customer-match)

### オプション C：クラウドストレージへのファイルベースのアクティベーション

**CRMのインポート、データウェアハウスのエンリッチメント、パートナーデータの共有、ファイルベースのハンドオフが推奨されるカスタム統合など、アカウントオーディエンスがダウンストリームでどのように消費されるかについて最大限の柔軟性が必要な組織に最適：**&#x200B;です。

**仕組み：**

このオプションは、アカウントオーディエンスをCSV、JSON、またはParquet ファイルとして、スケジュールされた頻度でクラウドストレージの宛先（[!DNL Amazon S3]、[!DNL Azure Blob Storage]、[!DNL Google Cloud Storage]、SFTP）に書き出します。 書き出しには、アカウント属性、関連する連絡先からの個人レベルのフィールド、オーディエンスメンバーシップのメタデータが含まれます。 ダウンストリームのシステムは、独自のインポートプロセスを通じてこれらのファイルを使用します。

ファイルベースのアクティベーションでは、書き出し形式、フィールド選択、スケジュール設定を最も自由に制御できます。 完全な書き出し（実行ごとにオーディエンススナップショットを完了）と増分エクスポート（前回の実行以降の変更のみ）の両方をサポートします。 このアプローチは、ターゲットシステムにネイティブな[!DNL RT-CDP]宛先コネクタがない場合、または組織がデータをターゲットシステムに読み込む前にデータを前処理または変換する必要がある場合によく使用されます。

**重要な考慮事項：**

- クラウドストレージインフラストラクチャ （[!DNL S3]、[!DNL Azure Blob]、[!DNL GCS]、またはSFTP）が必要です
- 下流のインポートプロセスを構築し、維持する必要があります
- ファイル形式（CSV、JSON、Parquet）は、下流のシステム要件と一致する必要があります
- 書き出しのスケジュールは、下流の処理ウィンドウと一致させる必要があります

**利点：**

- オーディエンスデータの使用方法における最大限の柔軟性
- ファイルを読み込むことができる任意のダウンストリームシステムをサポート
- 書き出し形式、フィールド、スケジュール設定の完全制御
- 単一の書き出しから複数のダウンストリームコンシューマーにサービスを提供できます
- フル モードと増分エクスポート モードをサポート

**制限：**

- すべてのオプションで最も高い待ち時間（バッチのみ）
- 下流のインポートワークフローの構築と維持が必要
- ネイティブ統合なし – 追加の開発作業が必要
- ファイル管理（クリーンアップ、アーカイブ）は別途処理する必要があります

- [Azure Blob Storageの宛先](https://experienceleague.adobe.com/en/docs/experience-platform/destinations/catalog/cloud-storage/azure-blob)

### オプション D:CRM システムへのストリーミングアクティベーション

**営業部門の可視化、担当領域の割り当て更新、または自動化された営業ワークフローのために、アカウントの選定の変更をほぼリアルタイムでCRM （[!DNL Salesforce]または[!DNL Dynamics]）に反映する必要がある営業とマーケティングの連携のユースケースに最適です。**

**仕組み：**

このオプションでは、ストリーミング宛先コネクタ （[!DNL Salesforce] CRM、[!DNL Microsoft Dynamics]）を使用して、アカウントオーディエンスメンバーシップの変更をCRM アカウントまたは連絡先レコードに直接プッシュします。 アカウントがオーディエンスの対象となる、またはオーディエンスから離脱すると、CRM レコードが、セグメントメンバーシップを示すカスタムフィールドで更新されます。 営業部門は、これらのフィールドにもとづいてCRM レポート、ビュー、アラートを作成できます。

オーディエンスメンバーシップの変更が発生すると、ストリーミングコネクタは増分更新を送信します。 フィールドマッピングは、[!DNL RT-CDP]個のアカウントと個人の属性をCRM フィールドに接続し、[!DNL RT-CDP]からの統合データを使用してCRM レコードを強化することを可能にします。 このアプローチにより、マーケティングインテリジェンス（アカウントの選定）とセールス実行（CRM ワークフロー）の間のループを閉じることができます。

**重要な考慮事項：**

- 適切な書き込み権限を持つCRM API アクセスが必要
- オーディエンスメンバーシップデータを受信するには、CRM カスタムフィールドを作成する必要があります
- ストリーミングボリュームは、CRM API レート制限内である必要があります
- フィールドの更新に対応するには、CRM ワークフローオートメーションを設定する必要があります

**利点：**

- セールス部門の状況を可視化するためのほぼリアルタイムのCRM更新
- オーディエンスの変更によってトリガーされる自動CRM ワークフローを有効にします
- [!DNL RT-CDP]からの統合アカウントデータでCRM レコードを強化します
- アカウントレベルと連絡先レベルの両方のフィールド更新をサポート

**制限：**

- CRM APIのレート制限により、大量のアップデートがスロットリングされる可能性がある
- CRMのカスタマイズが必要（カスタムフィールド、ワークフロー）
- ネイティブコネクタとして[!DNL Salesforce]と[!DNL Microsoft Dynamics]に制限されています
- ストリーミングの評価の制約は、基本的なオーディエンスに適用されます

- [Salesforce CRMの宛先](https://experienceleague.adobe.com/en/docs/experience-platform/destinations/catalog/crm/salesforce)
- [Microsoft Dynamics 365の宛先](https://experienceleague.adobe.com/en/docs/experience-platform/destinations/catalog/crm/microsoft-dynamics-365)

### 適切なオプションの選択

次の意思決定ガイダンスを活用して、適切なアクティベーションアプローチを選択します。

1. **主な目標がB2B リードナーチャリングで、[!DNL Marketo Engage]**&#x200B;を使用している場合は、オプション Aを選択します。ネイティブストリーミングコネクタにより、MA ワークフローの所要時間を短縮できます。

2. **主な目標が広告によるアカウントベースの認知度と需要創出である場合**、オプション Bを選択します。[!DNL LinkedIn] 一致オーディエンスは、最も強力なアカウントレベルのターゲティングを提供します。 広範なリーチのために他の広告プラットフォームを利用：

3. **ネイティブの[!DNL RT-CDP] コネクタ**&#x200B;を持たないシステムにアカウントオーディエンスをフィードする必要がある場合、またはデータ形式とダウンストリーム処理を最大限に制御する必要がある場合は、オプション Cを選択します。これは、パートナーとのデータ共有やデータウェアハウスの強化にも最適な選択肢です。

4. **主な目標がセールス支援とマーケティング適格アカウントのCRM表示である場合**、オプション Dを選択します。これにより、CRM レコードをほぼリアルタイムで更新することで、マーケティングとセールスの引き継ぎを実現できます。

多くのB2B企業は、ナーチャリングプログラムのオプション A、広告[!DNL LinkedIn]のオプション B、CRM同期のオプション Dなど、複数のオプションを同時に導入します。 これらのオプションは相互に排他的ではありません。同じアカウントオーディエンスを複数の宛先に同時にアクティベートできます。

## 実装フェーズ

### フェーズ 1：アカウントプロファイルの強化

この段階では、CRM、MA （マーケティングオートメーション）、サードパーティなどからのデータを統合することで、統合されたアカウントのプロファイルを構築します。

**アプリケーション関数：** [!DNL RT-CDP] B2B: アカウントプロファイルの統合、[!DNL RT-CDP] B2B: B2B ID解決

**設定する内容：**&#x200B;このフェーズでは、CRM、マーケティングオートメーション、サードパーティのソースからデータを統合することで、統合アカウントプロファイルを確立します。 B2BのID解決は、個人とアカウントの関係をマッピングすることで、個人レベルのエンゲージメントデータ（メール開封、web訪問、コンテンツのダウンロード）を集約し、アカウントレベルのオーディエンス評価に利用することができます。 このフェーズは、F2、F3、F4の基本機能に基づいています。これらの機能は、既に導入されている必要があります。

このフェーズの&#x200B;**決定ポイント：**

>[!NOTE]
>**決定：アカウント ID戦略**
>
>システムをまたいでプライマリアカウントキーとして機能する識別子はどれですか？
>
>| オプション | 選択するタイミング | 検討事項 |
>| --- | --- | --- |
>| CRM アカウント ID （例：[!DNL Salesforce] アカウント ID） | CRMは、アカウントの記録システムです | 最も一般的なアプローチ。 [!DNL RT-CDP]とCRMの整合性を確保します。 すべてのソースシステムが同じCRM アカウント IDを参照する必要があります。 |
>| カスタムアカウント ID | 複数のCRM インスタンスや専用のアカウントマスター | ソースシステム IDと正規アカウント IDの間にマッピングレイヤーが必要です。 より柔軟ですが、複雑さも増します。 |

### フェーズ 4：オーディエンスのアクティベーション

このフェーズでは、評価されたアカウントオーディエンスを、設定された宛先に公開します。

**アプリケーション関数：** [!DNL RT-CDP] B2B: アカウント Audience Activation、[!DNL RT-CDP]: Audience Activation

**設定する内容：**&#x200B;このフェーズでは、評価されたアカウントオーディエンスを設定された宛先に公開します。 アクティベーションにより、アカウントオーディエンス（ソース）と外部宛先（ターゲット）を接続するデータフローが作成され、属性マッピングが適用され、設定されたスケジュールまたはストリーミング動作に従って書き出しが開始されます。 また、除外オーディエンスを設定して、対象外のアカウントをアクティベーションから除外します。

このフェーズの&#x200B;**決定ポイント：**

>[!NOTE]
>**決定：エクスポートの種類**
>
>アクティベーションでは、完全なオーディエンススナップショットを書き出すか、増分変更のみを書き出すか？
>
>| オプション | 選択するタイミング | 検討事項 |
>| --- | --- | --- |
>| 増分書き出し | ストリーミング宛先（[!DNL Marketo]、CRM）または下流システムが差分を処理するバッチ宛先 | 書き出しあたりのデータ量が少ない。 より迅速な処理： ステートを管理するためにダウンストリームシステムが必要です。 ストリーミング宛先のデフォルト。 |
>| 完全書き出し | ダウンストリームシステムが実行ごとに完全なオーディエンスを置き換えるバッチ宛先 | データ量は多くなりますが、下流ロジックはシンプルです。 ダウンストリームシステムが状態を維持しない場合に便利です。 ファイルベースの宛先で使用できます。 |

>[!NOTE]
>**決定：アクティベーションの範囲**
>
>宛先ごとにアクティブ化するオーディエンスの数はどれくらいですか？
>
>| オプション | 選択するタイミング | 検討事項 |
>| --- | --- | --- |
>| データフローあたりの単一オーディエンス | オーディエンスと宛先の明確なマッピングによるシンプルなアクティベーション | 監視とトラブルシューティングが容易： 1つのオーディエンスの失敗が他のオーディエンスに影響を与えることはありません。 |
>| データフローごとの複数オーディエンス | 複数の関連オーディエンスが同じ宛先に向かっている | 宛先接続のより効率的な利用： すべてのオーディエンスは、同じフィールドマッピングとスケジュールを共有しています。 |

**UI ナビゲーション：**&#x200B;接続/宛先/参照/宛先を選択/オーディエンスをアクティブ化

**キー設定の詳細：**

- オーディエンスリストからターゲットオーディエンスを選択します
- フェーズ 3で設定されたフィールドマッピングを確認し、確認します
- バッチ宛先の場合：書き出しスケジュール（時間、頻度）を設定します
- ストリーミング宛先の場合：設定直後にアクティベーションが開始されます
- オプションで、「除外」機能を使用して抑制オーディエンスを追加します
- データフローを検証するための最初のテストアクティベーション実行をトリガーする

**オプションが異なる場所：**

オプション A （[!DNL Marketo Engage] ストリーミング）の&#x200B;**:**

アクティベートするアカウントオーディエンスを選択します。 アクティベーションはすぐにストリーミングを開始します。 [!DNL Marketo]で、セグメント メンバーシップ フィールドを使用してリード/連絡先レコードが更新されていることを確認します。 これらのフィールドの変更に基づいて、トリガーする[!DNL Marketo] スマートキャンペーンを設定します。

**オプション B （Advertising Platform バッチ）の場合：**

アカウントオーディエンスを選択し、毎日の書き出しスケジュールを設定します。 最初の書き出しが完了したら、広告プラットフォームで、オーディエンスが表示され、メンバー数が入力されていることを確認します。 プラットフォームが最初のオーディエンスファイルを処理するのに24～48時間かかります。

**オプション C （クラウドストレージファイルベース）の場合：**

アカウントオーディエンスを選択し、書き出しスケジュールとファイル形式を設定します。 最初の書き出し後、ファイルが想定される形式とコンテンツでターゲットストレージの場所に表示されることを確認します。 ダウンストリームのインポートプロセスが、書き出されたファイルを正常に使用することを確認します。

### フェーズ 5：ガバナンスとモニタリング

この段階では、アカウントのオーディエンスのアクティベーションがデータガバナンスポリシーと同意の設定に準拠し、継続的なアクティベーションデータフローの正常性が監視されます。

**アプリケーション関数：** [!DNL RT-CDP] B2B: B2B データガバナンス、[!DNL RT-CDP]：同意とガバナンスの適用

**設定する内容：**&#x200B;このフェーズでは、アカウントオーディエンスのアクティベーションがデータガバナンスポリシーと同意設定に準拠し、継続的なアクティベーションデータフローが正常性のために監視されていることを確認します。 B2B データガバナンスでは、機密性の高いアカウント属性（収益、サードパーティプロバイダーからの従業員数）に対して制限を適用し、同意の履行では、個人レベルのコミュニケーションでオプトアウトの設定を尊重します。 監視は、アクティベーションデータフローが正常に完了していることを確認します。

### ガードレールと制限

このユースケースパターンに適用される次のプラットフォームのガードレールと制限を確認します。

- アカウントオーディエンスを含む、サンドボックスごとに最大4,000個のセグメント定義 – [&#x200B; セグメント化ガードレール &#x200B;](https://experienceleague.adobe.com/en/docs/experience-platform/profile/guardrails)
- アカウントオーディエンスは、主にバッチ評価を使用して評価されます。ストリーミングの実施要件は、単純なアカウント属性条件に限定されます
- 宛先接続あたり最大100 データフロー – [宛先ガードレール &#x200B;](https://experienceleague.adobe.com/en/docs/experience-platform/destinations/guardrails)
- バッチ宛先は、ファイルセグメントごとに最大500万プロファイルを書き出します
- ストリーミング宛先には、宛先パートナーによって設定された1秒あたりのスループット制限（例：[!DNL Marketo] API レート制限）があります
- 構成されたオーディエンス（オーディエンス構成から）はバッチ評価に限定され、ストリーミングを使用できません
- オーディエンス構成キャンバスごとに最大10個の構成ブロック
- [!DNL LinkedIn]件の一致したオーディエンスをアクティブ化するには、最小オーディエンスサイズ（通常は300人）が必要です
- CRM ストリーミング宛先は、CRM プロバイダーのAPI レート制限（例：[!DNL Salesforce] バルク APIの日別制限）の対象となります。
- [!DNL RT-CDP] B2B edition ライセンスは、法人アカウントプロファイルの合計数を管理します – [RT-CDP製品の説明](https://helpx.adobe.com/legal/product-descriptions/real-time-customer-data-platform-b2b-edition-prime-and-ultimate-packages.html)

### よくある落とし穴

このユースケースパターンを実装する際には、次の一般的な問題に注意してください。

- **不完全な個人とアカウントのマッピング：**&#x200B;個人レコード（リード、取引先責任者）がB2B ID解決を通じてアカウントレコードに正しくリンクされていない場合、個人の属性またはアクティビティに依存するアカウントオーディエンスは、適格なアカウントを下回ります。 アカウントのオーディエンスを作成する前に、F4で個人とアカウントの関係を検証します。

- **古いCRM データが原因でオーディエンスドリフト：** CRM ソースコネクタが定期的なスケジュールで実行されていないか、サイレントで失敗している場合、アカウント属性（業界、収益、ステータス）が古くなります。 これにより、オーディエンスには、資格がなくなったアカウントや、資格を得る必要があるアカウントを除外するアカウントが含まれます。 ソースコネクタデータフローの正常性を監視します。

- **Advertising プラットフォームの一致率の期待値：** [!DNL LinkedIn]以外の広告プラットフォームに対してアカウントオーディエンスをアクティブ化する場合、一致率は、対象アカウントに関連付けられた連絡先に有効な個人レベルのID （ハッシュ化されたメール）を持つかどうかに応じて異なります。 メールアドレスを持つ連絡先が関連付けられていないアカウントは一致しません。 マッチ率を監視し、連絡先データの強化を検討する。

- **[!DNL Marketo]フィールドマッピングの不整合：** [!DNL Marketo Engage]にストリーミングする場合、[!DNL RT-CDP] フィールドマッピングは既存の[!DNL Marketo] リードまたは連絡先フィールドをターゲットにする必要があります。 マッピングされた[!DNL Marketo] フィールドが存在しない場合、更新はサイレント フェイルになります。 宛先を設定する前に、[!DNL Marketo]のすべてのターゲットフィールドを事前作成します。

- **ガバナンスポリシーブロックのアクティベーション：** アカウント属性フィールド（特にサードパーティの企業特性データ）のデータ使用ラベルは、広告宛先にアクティベートする際にガバナンス違反をトリガーする場合があります。 アクティベートする前にガバナンスコンプライアンスを評価し、フィールドマッピングを調整して、必要に応じて制限されたフィールドを除外します。

- **アカウントと個人データをバッチのみの評価と組み合わせたアカウントオーディエンス：**&#x200B;個人レベルの行動イベントを参照するアカウントオーディエンス（例：「少なくとも1人の連絡先が過去30日間にメールを開封したアカウント」）には、バッチ評価が必要です。 ユースケースでリアルタイムの選定が想定されている場合、この制約により予期しない遅延が発生する可能性があります。

### ベストプラクティス

最適な結果を得るには、次の推奨事項に従ってください。

- 少数の明確に定義されたアカウントオーディエンス（ICP層、業界垂直層、エンゲージメント層）から始めてから、複雑な多属性定義に拡張する
- 既存顧客、アクティブな機会、最近エンゲージしたアカウントに対して専用の抑制オーディエンスを作成し、無駄な支出やチャネルの競合を回避します
- オーディエンス構成を使用して、集計されたエンゲージメントスコアでアカウントをランク付けすることで、階層化されたアカウントオーディエンス（高/中/低エンゲージメント）を構築します
- 同じアカウントオーディエンスを、複数の宛先に対して同時にアクティベートして、調整されたマルチチャネルキャンペーンを行います（例：広告用に[!DNL LinkedIn]、メール用に[!DNL Marketo]、セールス用にCRMを表示）。
- ターゲティング基準と対象となるチャネルを含む、アカウントオーディエンスに対して一貫した命名規則（例：「B2B_ICP_Enterprise_Tech_LinkedIn」または「B2B_Suppression_ActiveOpps」）を実装する
- オフピーク時間中にバッチアクティベーションをスケジュールし、下流システムへの影響を最小限に抑え、広告プラットフォームの処理ウィンドウと連携させます
- 最初のアクティベーション後、宛先ごとにマッチ率を監視し、ID フィールドマッピングを繰り返してマッチングを向上させます
- [!DNL Marketo Engage]で双方向データフローを維持：[!DNL Marketo]から[!DNL RT-CDP] （ソースコネクタ）にエンゲージメントデータを取り込み、クローズドループシステム用に[!DNL Marketo] （宛先コネクタ）にオーディエンスをアクティブ化します

### トレードオフの決定

導入の意思決定をおこなう際には、次のトレードオフを考慮してください。

>[!NOTE]
>**トレードオフ：オーディエンスの鮮度と評価の複雑さ**
>
>アカウント属性と個人レベルの行動データを組み合わせたアカウントオーディエンスは、最も正確なターゲティングを提供できますが、バッチ評価（日次リフレッシュ）に限定されます。 アカウントの属性にもとづいてオーディエンスを簡素化するだけで、ほぼリアルタイムに更新され、ストリーミング評価の対象となる可能性があります。
>
>- **バッチ（複雑な条件）のお気に入り：**&#x200B;精度のターゲティング、企業特性シグナルと行動シグナルの組み合わせ、包括的なアカウントスコアリング
>- **ストリーミング（単純な条件）の好み：** オーディエンスの更新スピード、アカウントの変更に対するリアルタイムの対応、[!DNL Marketo]およびCRMでのアクションまでの時間の短縮
>- **推奨事項：**&#x200B;毎日の更新が許容されるプライマリターゲティングオーディエンスに対するバッチ評価の使用（ほとんどのB2B ユースケース）。 アカウントステータスの変更や価値の高いアカウントアラートなど、時間の制約を受けるトリガーについては、ストリーミング評価を予約します。

>[!NOTE]
>**トレードオフ：一元化されたアクティベーションと宛先固有のオーディエンス**
>
>単一の統合アカウントオーディエンスを複数の配信先で活用したり、各チャネルの機能や要件に合わせて配信先固有のオーディエンスを作成したりできます。
>
>- **一元化されたオーディエンスのお気に入り：** チャネル間での一貫性、よりシンプルなオーディエンス管理、統合レポート
>- **宛先固有のオーディエンスのお気に入り：** チャネルごとの最適化されたターゲティング （[!DNL LinkedIn]は企業レベルの属性からメリットを受けるが、[!DNL Marketo]はリードレベルの詳細が必要な場合など）、チャネル固有の抑制ルール
>- **推奨事項：**&#x200B;一貫性を保つために一元化されたオーディエンスから開始し、チャネルの要件が大幅に異なる場合（広告とメールに対して異なる抑制ウィンドウなど）にのみ、宛先固有のバリエーションを作成します。

>[!NOTE]
>**トレードオフ：リアルタイム CRM同期とバッチ CRM更新の比較**
>
>CRMにストリーミングすると、セールスがすぐに可視化されますが、CRM APIの割り当てが消費されます。 バッチファイルの書き出しは効率的ですが、遅延が発生します。
>
>- **ストリーミング CRM同期のメリット：**&#x200B;営業対応、即時のアカウントアラート、リアルタイムのパイプラインの可視化
>- **CRMのバッチ更新のメリット：** API割り当て量の保存、一括更新の効率性、CRM データ読み込みウィンドウとの整合性
>- **推奨事項：**&#x200B;優先度の高いアカウント選定の変更にストリーミング CRM同期を使用する（アカウントが「営業準備完了」層に移動するなど）。 毎月のアカウントスコアリングの更新や、地域の再割り当てリストなどの一括更新には、バッチファイルの書き出しを使用できます。
