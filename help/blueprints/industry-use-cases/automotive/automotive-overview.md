# 自動車のユースケース

自動車関連企業は、Adobe Experience Platformを使用して、販売店とのやり取り、オンラインでの車両調査、サービスレコード、コネクテッドカーシステムに関する顧客データを、各オーナーの単一のビューに統合しています。 この基盤により、最初の車両調査から購入、サービス、ロイヤルティに至るまで、所有権のライフサイクル全体を通じてパーソナライズされたエクスペリエンスが可能になります。

## ユースケース別の技術的な考慮事項

### 車両購入ジャーニーのパーソナライゼーション

- ディーラー管理システムからの車両在庫データは、近くのディーラーで実際に利用可能であることを反映するように、頻繁に統合および更新する必要があります。
- 車両設定アクティビティ、比較ツールの使用状況、パンフレットのダウンロードなど、web サイト全体にわたる顧客調査の行動を取り込んで、購入意図を正確に特定する必要があります。
- リードスコアリングモデルでは、オンラインエンゲージメントと、ディーラーへの訪問や電話での問い合わせなどのオフライン信号の両方を考慮して、最も意図の高い見込み客に優先順位を付ける必要があります。
- 見込み客 [!DNL Real-Time Customer Data Platform] 登録するか販売店に訪問したら、プロファイルで匿名の web リサーチセッションを既知の顧客 id と結合する必要があります。

### サービスの予定のリマインダー

- コネクテッドカーシステムまたはディーラーサービスレコードからの車両の走行距離やテレマティクスデータは、適切なサービス間隔でトリガーリマインダーに顧客プロファイルに流れる必要があります。
- リマインダーメッセージには、予約の摩擦を最小限に抑えるために、顧客の車両情報と推奨されるサービス項目を事前入力するワンクリックスケジュールリンクを含める必要があります。
- 最近完了したサービスの推奨を避け、期日の特定のメンテナンス項目でリマインダーをパーソナライズするには、サービス履歴レコードを統合する必要があります。
- メッセージのタイミング [!DNL Journey Optimizer]、ディーラーサービスの処理能力と営業時間を考慮して、顧客がリマインダーを受け取ったときに予約できるようにしてください。

### 下取り価格キャンペーン

- お客様と共有する下取り見積りが正確で競争力を持つようにするには、サードパーティプロバイダーからの車両評価データを定期的に統合および更新する必要があります。
- 所有権ライフサイクルモデルでは、最適なアウトリーチウィンドウを特定するために、リースの有効期限、ローンのペイオフタイムライン、車両セグメント別の平均所有期間などの要因を考慮する必要があります。
- Campaign メッセージでは、顧客の現在の車両を動的に参照し、顧客の好みや予算に合った特定のアップグレードオプションを提案する必要があります。
- [!DNL Experience Platform] セグメントでは、時間不足によるアウトリーチを避けるために、車両を最近購入した顧客や、現在アクティブなサービスに関する紛争に参加している顧客を除外する必要があります。

### パーツおよびアクセサリの推奨事項

- 製品カタログデータには車両互換性情報を含める必要があり、レコメンデーションでは、顧客の特定の車両年度、車種、モデルに適合するパーツやアクセサリーのみが表示されるようにする必要があります。
- 季節的および地域的な要因が推奨事項に影響を与える必要があります。寒い気候では冬のアクセサリーを宣伝する必要がありますが、アクティブな愛好家のコミュニティでは、パフォーマンスアップグレードの方が地域に響く可能性があります。
- 部品およびアクセサリーページのブラウジング動作をキャプチャして顧客プロファイルに関連付け、提示された関心に基づいてレコメンデーションを絞り込む必要があります。
- Web SDK[!DNL Experience Platform] 実装では、互換性フィルタリングが正確であることを確認するために、認証済みセッションから車両識別番号データを取得する必要があります。

### 車両リコール通知

- リコール通知がすべての影響を受ける所有者に届き、影響を受けないお客様に送信されないように、車両識別番号レコードを顧客プロファイルで正確に保持する必要があります。
- リコールメッセージは、該当する各市場における安全通知の内容、タイミング、配信確認に関する規制要件に準拠する必要があります。
- セーフティクリティカルな通信では、マーケティングメッセージよりも高い配信保証が必要となるため、リーチを最大化するには、マルチチャネル配信（メール、テキストメッセージ、プッシュ通知、物理的なメール）を使用する必要があります。
- [!DNL Journey Optimizer] は、リコールの応答状況を個々のレベルで追跡し、定義された期間内にサービスをスケジュールしていないオーナーのフォローアップ通信をエスカレーションする必要があります。

### 新しいモデルのローンチキャンペーン

- ローンチキャンペーンのオーディエンスセグメントでは、最も傾向が高い見込み客を特定するために、現在の車両モデル、所有期間、過去のモデルの興味シグナルおよび人口統計学的な適合性を考慮する必要があります。
- ローンチコンテンツは、顧客の現在の手段を参照し、可能性の高い優先事項に対処する特定の改善点や機能をハイライトするためにパーソナライズする必要があります。
- キャンペーンのタイミングは、市場に適したタイミングで顧客が情報を受け取れるように、禁輸日および地域の立ち上げスケジュールと調整する必要があります。
- [!DNL Real-Time Customer Data Platform] audience activation では、所有チャネルのアウトリーチと連携して有料メディアをサポートするために、ローンチセグメントを広告プラットフォームに同期する必要があります。

### 融資および保険オファー

- 融資規制に準拠するように慎重に金融オファーの実施要件ルールを設定し、顧客に提示されるオファーが、実際に適格なものになるようにする必要があります。
- 財務情報は高いプライバシーと規制要件の対象となるため、信用プロファイルデータの統合には、安全な処理と厳格なアクセス制御が必要です。
- オファーの提示は、各適用市場の消費者金融規制に準拠して条件、料金、条件を明確に開示する必要があります。
- 決定ルールでは、単に率でランク付けするのではなく、関連度でオファーをランク付けするために、車両価格、頭金、ローン期間の環境設定を考慮する必要が [!DNL Journey Optimizer] ります。

### テストドライブのスケジュール

- ディーラー在庫システムを統合して、お客様が興味を持っている特定の車両モデルとトリミングが、推奨される販売店でのテストドライブに利用可能であることを確認する必要があります。
- ロケーションベースのディーラーの推奨事項では、顧客の住所、ディーラーの評価、現在のアポイントメントの空き時間を考慮して、最も便利なオプションを提案する必要があります。
- 試験運転の確認とリマインダーのメッセージには、指示、ディーラーの連絡先情報、および予想される内容を含め、不在率を減らす必要があります。
- 行動データ [!DNL Experience Platform]、まだ準備ができていない初期段階の研究者への早期アウトリーチを避け、テストドライブを提案する最適な瞬間を特定する必要があります。

### 所有者ロイヤルティプログラム

- ロイヤルティ層の計算には、車両購入履歴だけでなく、サービス訪問、パーツ購入、紹介、イベント出席など、複数のエンゲージメントのディメンションを組み込む必要があります。
- 販売店やサービスセンターでの報酬履行システムを統合して、サービスのポイントでロイヤルティのメリットをシームレスに引き換えられるようにする必要があります。
- コミュニケーションは、所有権のライフサイクルステージに基づいて適応する必要があり、新しい所有者には、最初の 1 年で異なる価値提案を提供しますが、長期的な所有者には、潜在的なアップグレードに近づく必要があります。
- ジャーニーロジック [!DNL Journey Optimizer]、顧客がロイヤルティレベル間を移動した際に、リアルタイムで階層の変化を検出し、お祝いメッセージや再エンゲージメントメッセージをトリガーする必要があります。

### 保証および拡張サービスプラン

- 通常は有効期限の 60～90 日前に、お客様のプロファイルで保証期限の日付と保証期間の詳細を正確に追跡して、適切なタイミングでトリガーをサポートする必要があります。
- ハイマイレージのドライバーは、低マイレージのオーナーとは異なるカバレッジの恩恵を受けるので、車両の走行距離およびコネクテッドカーのデータまたはサービスレコードからの使用パターンは、プランの推奨事項を通知する必要があります。
- 計画の比較コンテンツは明確で、専門用語を使用しないことが必要です。顧客が、カバーしている内容と、潜在的な自己負担修理コストに対する価値を正確に理解するのに役立ちます。
- [!DNL Real-Time Customer Data Platform] のセグメントは、工場保証の有効期限が切れている顧客、既存の延長プランが更新間近の顧客、現在の保証がない顧客を区別する必要があります。

### コネクテッドカー機能の有効化

- コネクテッドカープラットフォームデータを統合して、各車両で利用可能な機能と、所有者が既にアクティブ化または使用した機能を特定する必要があります。
- 機能の採用トラッキングでは、アクティベーションイベント、使用頻度およびエンゲージメントの深さをキャプチャして、機能レコメンデーションのシーケンスとペーシングをパーソナライズする必要があります。
- 車載通知チャネルは、メールやアプリの通知と連携して、出張が検出された際のナビゲーション機能の提案など、最も関連性の高いタイミングで機能プロンプトを配信する必要があります。
- プロファイル [!DNL Experience Platform]、インストール済みパッケージやソフトウェアバージョンなどの車両技術設定データを保存して、機能レコメンデーションが所有者の特定の車両に適合していることを確認する必要があります。

### ディーラーネットワークの調整

- ディーラー在庫フィードを頻繁に統合および更新して、顧客に表示される車両の可用性が各ディーラーのロットの内容を正確に反映できるようにする必要があります。
- 顧客からディーラーへの割り当てロジックでは、近接性、ディーラーの特殊性、言語の好み、および既存のディーラーの関係を考慮して、最適な一致を提供する必要があります。
- リードのルーティングルールでは、顧客がオンラインで購入の興味を表明した場合、問い合わせが顧客のリサーチ活動に関する完全なコンテキストを持つ適切なディーラーにすばやく到達することを確認する必要があります。
- id 解決 [!DNL Experience Platform]、顧客が複数のディーラーとやり取りするシナリオに対処し、各ディーラーが独自の顧客関係を把握しながら、統一されたプロファイルを維持する必要があります。
