# テクノロジーのユースケース

ハイテク企業は、Adobe Experience Platformを使用して、web、モバイル、製品サーフェスからのデータ収集を一元化し、リアルタイムのイベントを、製品やマーケティングプログラムを強化する分析、データウェアハウス、アクティベーションの宛先に配信しています。 テクノロジー部門は、エッジでのイベント収集を統合することで、クライアントサイドの複雑さを軽減し、データ品質を向上させ、あらゆる下流システムが単一の信頼できるソースから一貫した行動データを受け取れるようにします。

## リアルタイムのイベント転送

Edge Networkを通じて収集したリアルタイムの行動イベントを、分析ツール、データウェアハウス、パートナーツールに転送し、エンリッチメントとアクティベーションを実現します。 エッジでイベント収集を一元化し、複数の宛先に転送することで、クライアントサイドのタグオーバーヘッドを削減し、データ品質を向上させ、すべてのダウンストリームシステムが単一の信頼できるソースから一貫したイベントデータを受け取ることができます。

### ビジネスへの影響

リアルタイムのイベント転送を導入しているテクノロジー企業は、分析、データウェアハウス、アクティベーション宛先をまたいでイベントデータの一貫性を向上させながら、クライアントサイドのタグ読み込みと関連するパフォーマンスのオーバーヘッドを低減します。 また、サーバーサイド転送は、ページの重み付けを軽減し、読み込みパフォーマンスを向上させ、ユーザーエクスペリエンス指標に直接利益をもたらします。

### 導入方法

[&#x200B; イベント転送](/help/blueprints/use-case-patterns/audience-building-activation/event-forwarding.md) パターンを使用して、Adobe Experience Platform Web SDKで収集したイベントをEdge Network経由で設定されたサーバーサイドの宛先にルーティングします。 これは、クライアントサイドの宛先ごとに個別のタグを管理するのではなく、単一の収集ポイントからサーバー間でイベントを配信することを目的としている場合に適したパターンです。これにより、ページの重みが増し、システム間でデータの一貫性が損なわれます。

### 技術的な考慮事項

- Edge Network イベント転送では、イベント収集をAdobe Experience Platform Web SDKまたはMobile SDKに移行する必要があります。転送先を設定する前に、既存のタグベースの実装の互換性を評価する必要があります。
- 転送ルールは、各宛先で必要なフィールドのみを送信するように設定する必要があります。これにより、データ転送コストが増加し、コンプライアンスの露出が発生するため、完全なXDM ペイロードを、フィールドの小さなサブセットしか必要としない宛先に転送することは避けてください。
- 宛先コネクタは、エラーを適切に処理する必要があります。イベント転送パイプラインは、障害時のデータ損失を防ぐために、宛先に対して再試行ロジックとアラートを実装する必要があります。
- 転送先ごとにデータガバナンスポリシーを確認し、エッジでキャプチャされたユーザーの同意設定が転送設定で尊重されるようにする必要があります。
