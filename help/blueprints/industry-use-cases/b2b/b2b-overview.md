# B2B のユースケース

B2B 企業は、Adobe Experience Platformを使用してアカウントおよびユーザーレベルのデータを統合することで、マーケティングチームやセールスチームが購入ジャーニーのすべてのステージにおいて、調整された関連性の高いエクスペリエンスを提供できるようになります。 パイプラインの高速化から顧客の拡張に至るまで、これらのユースケースは、B2B チームが複雑なデータを測定可能なビジネス成果に変換する方法を示しています。

>[!NOTE]
>
>アカウントベースのアクティベーションおよび購入グループ管理を含む、B2B 固有のアーキテクチャブループリントについては、[B2B アクティベーションおよびマーケティングブループリント ](/help/blueprints/b2b/overview.md) を参照してください。

## Account-Based MarketingPersonalization

アカウントプロファイル、エンゲージメント履歴、購入シグナルに基づいて、ターゲットアカウントのマーケティングコミュニケーションとコンテンツをパーソナライズします。 メッセージを各アカウントの固有のコンテキストに合わせることで、マーケティングチームは問題を解決し、適切な関係者を適切なコンテンツに関与させることができます。

### ビジネスへの影響

アカウントベースのマーケティングパーソナライゼーションを実装する組織では、通常、アカウントエンゲージメント率が 30～40% 向上し、より強力なパイプラインと契約の進行の高速化を促します。

### 実装方法

[B2B Audience Activation](/help/blueprints/use-case-patterns/audience-building-activation/b2b-audience-activation.md) パターンを使用して、アカウントレベルのオーディエンスを作成し、チャネルをまたいでパーソナライズされたコンテンツをアクティブ化します。 このパターンは、アカウントベースの戦略専用であり、アカウントとユーザーレベルの両方のターゲティングをサポートします。

### 技術上の考慮事項

- CRM データ（[!DNL Salesforce] や [!DNL Microsoft Dynamics] など）を統合して、最新のアカウント階層、商談ステージ、所有権割り当てを維持します。
- アカウントレベルのプロファイルスキーマを設定して、業界、従業員数、収益範囲、テクノロジースタックなどの基本属性を取り込みます。
- 個人とアカウントの関係をマッピングして、個々のエンゲージメントのシグナルがアカウントレベルのスコアリングおよびセグメント化にロールアップされるようにします。
- [!DNL Marketo Engage] と連携して、マーケティング自動化キャンペーンをプラットフォーム主導のアカウントオーディエンスと調整します。


## リードのスコアリングと育成

プロファイルデータと行動に基づいてリードを自動的にスコアリングし、高スコアリングのリードを他のユーザー向けにパーソナライズされた育成キャンペーンで販売にルーティングします。 このアプローチにより、セールスチームは最も有望な機会に集中できる一方で、マーケティング部門は初期段階の見込み客を継続的に開発できます。

### ビジネスへの影響

行動リードスコアリングと自動育成を実装する企業は、通常、リードトゥオポチュニティコンバージョンが 25～35% 向上し、パイプラインの速度が加速し、セールスの生産性が向上します。

### 実装方法

[ マルチステップオーケストレーションされたジャーニー](/help/blueprints/use-case-patterns/campaign-management-orchestration/multi-step-orchestrated-journey.md) パターンを使用して、リードスコアの変化と行動のトリガーに対応するブランチ育成ジャーニーを設計します。 このパターンは、育成トラックと販売ハンドオフワークフローの間でリードをルーティングするために必要な条件付きロジックをサポートしています。

### 技術上の考慮事項

- マーケティングシステムと販売システムの間でリードスコアと選定ステータスの一貫性を維持するために、双方向の CRM 同期（[!DNL Salesforce] や [!DNL Microsoft Dynamics] など）を設定します。
- デモグラフィック （役割、会社の規模、業界）と行動（コンテンツのダウンロード、ウェビナーの出席、製品ページの訪問回数）のスコアリングディメンションの両方を定義します。
- プラットフォーム間でスコアの競合が発生しないように、リードのスコアリングモデルと [!DNL Marketo Engage] を調整します。
- スコア減衰ルールを設定して、処理が停止するリードの優先順位を経時的に下げます。


## 見込み客向けのコンテンツPersonalization

見込み客の業界、役割、会社の規模、エンゲージメント履歴に基づいて、web サイトのコンテンツ、リソース、オファーをパーソナライズします。 見込み客が特定の状況に関連するコンテンツを見つけると、より深く関わり合い、funnelをより迅速に進めます。

### ビジネスへの影響

既知の見込み客に対して web エクスペリエンスをパーソナライズする B2B 組織では、通常、コンテンツエンゲージメントが 20～30% 増加し、より適格なパイプラインとセールスサイクルの短縮につながります。

### 実装方法

[ 既知の訪問者 web / アプリPersonalization](/help/blueprints/use-case-patterns/personalization/known-visitor-web-app-personalization.md) パターンを使用して、統合された見込み客プロファイルに基づいてカスタマイズされたコンテンツエクスペリエンスを提供します。 このパターンでは、リアルタイムプロファイルデータを活用して、各訪問者に最も関連性の高いリソース、導入事例およびアクションの呼び出しを提供します。

### 技術上の考慮事項

- ID 解決を実装して、認証またはフォームの送信後に、匿名の閲覧行動を既知の見込み客レコードに一致させます。
- アカウントレベルの属性（業界、セグメント、取引ステージ）と個人レベルの属性（役割、過去のコンテンツ使用）に基づいてパーソナライゼーションルールを定義します。
- をコンテンツ管理システムと統合して、ヒーローバナー、リソースのレコメンデーション、アクションの呼び出しを動的に入れ替えます。
- 完全 [!DNL Marketo Engage] 行動画像については、web トラッキングデータが統合プロファイルにフィードされることを確認します。


## イベントの登録とフォローアップ

イベントタイプと参加者プロファイルに基づいて、パーソナライズされたイベント登録の確認、リマインダーおよびイベント後のフォローアップを自動化します。 これにより、マーケティングチームを手動での調整から解放しながら、見込み客はイベントの前後、中、後にエンゲージし続けます。

### ビジネスへの影響

自動化されたパーソナライズされたイベント通信ワークフローにより、通常、イベントの出席率が 40～50% 向上し、イベントへの投資回収率が最大化され、見込み客との関係が強化されます。

### 実装方法

[ マルチステップオーケストレーションジャーニー](/help/blueprints/use-case-patterns/campaign-management-orchestration/multi-step-orchestrated-journey.md) パターンを使用すると、登録からポストイベント育成まで、イベントライフサイクル全体をオーケストレーションできます。 このパターンは、時間ベースのトリガー、イベントタイプ別の条件付きブランチ、複数チャネルのフォローアップシーケンスをサポートします。

### 技術上の考慮事項

- ウェビナープラットフォーム（[!DNL ON24]、[!DNL Zoom Webinar] など）の登録データと対面イベントシステムを統合プロファイルに統合します。
- 出席およびエンゲージメントのシグナル（セッション期間、質問、ポーリング応答）をキャプチャして、フォローアップメッセージをパーソナライズします。
- プログラムのステータスに合わせてイベントジャーニー [!DNL Marketo Engage] 調整し、システム間で一貫したレポートを維持します。
- 参加した登録者と参加しなかった登録者で異なるジャーニー分岐を設計し、それぞれに合わせたコンテンツを用意します。


## 製品体験版コンバージョンキャンペーン

パーソナライズされた製品のレコメンデーション、トレーニングリソースおよびオファーを使用して体験版ユーザーを参加させ、有料プランへのコンバージョンを促します。 製品探索を行っている体験版ユーザーに会うことで、チームは摩擦を取り除き、購入までの道のりを加速できます。

### ビジネスへの影響

パーソナライズされたトライアルコンバージョンキャンペーンを導入する組織では、通常、トライアルから有料へのコンバージョンが 25～35% 増加し、新しいビジネス収益に直接影響し、顧客獲得コストが削減されます。

### 実装方法

[ マルチステップのオーケストレーションされたジャーニー](/help/blueprints/use-case-patterns/campaign-management-orchestration/multi-step-orchestrated-journey.md) パターンを使用して、体験版ユーザー向けの時間ベースおよび行動ベースのコンバージョンジャーニーを構築します。 このパターンは、製品の使用マイルストーンに基づく条件付きパスをサポートし、適切なタイミングでターゲットを絞った微調整を可能にします。

### 技術上の考慮事項

- 製品の使用状況のテレメトリ（機能の採用、ログイン頻度、製品滞在時間）を取り込んで、リアルタイムの体験版エンゲージメントプロファイルを作成します。
- 使用状況ベースのマイルストーン（最初に作成されたプロジェクト、使用された主な機能、招待されたコラボレーションなど）をジャーニーのトリガーとして定義し、タイムリーにアウトリーチします。
- 試用版のステータスとコンバージョンイベントを [!DNL Salesforce] 以 [!DNL Microsoft Dynamics] に同期し、営業担当者が試用版アクティビティを完全に可視化できるようにします。
- [!DNL Marketo Engage] 育成プログラムと連携して、体験期間中にコミュニケーションが重複するのを避けます。


## カスタマーサクセスとオンボーディング

購入した製品と顧客プロファイルに基づいて、関連するトレーニング、リソース、サポートを利用して、顧客のオンボーディングジャーニーをパーソナライズします。 効果的なオンボーディングにより、価値実現までの時間が短縮され、長期的な顧客維持と成長の基盤が確立されます。

### ビジネスへの影響

パーソナライズされたオンボーディングプログラムを利用する組織では、通常、最初の 90 日以内に機能の採用が 50～60% 増加し、リテンションと拡張の収益の増加に直接貢献します。

### 実装方法

[ マルチステップオーケストレーションされたジャーニー](/help/blueprints/use-case-patterns/campaign-management-orchestration/multi-step-orchestrated-journey.md) パターンを使用して、製品、プラン層および顧客セグメントに合わせてカスタマイズされたオンボーディングシーケンスを調整します。 このパターンは、マイルストーンベースの進行をサポートし、オンボーディングの各段階で顧客が適切なガイダンスを受けられるようにします。

### 技術上の考慮事項

- オンボーディングマイルストーン（最初のログイン、初期設定、最初のワークフローが完了）をトラッキングするための製品使用状況テレメトリを取り込み、それに応じて次のジャーニーステップをトリガーします。
- 購入した製品、プラン層、ライセンスを取得したユーザーの数に基づいて、顧客プロファイルを正しいオンボーディングトラックにマッピングします。
- カスタマーサクセスプラットフォームデータを統合して、ヘルススコアを表示し、リスクのあるアカウントにフラグを付けて、プロアクティブな介入を実現します。
- オンボーディングステータスと採用指標を [!DNL Salesforce] 以 [!DNL Microsoft Dynamics] に同期させて、カスタマーサクセスマネージャーがアウトリーチを優先できるようにします。


## 契約更新キャンペーン

パーソナライズされたオファー、使用状況のインサイト、更新のインセンティブを使用して、契約更新に近づいているお客様を積極的に引き付けます。 タイムリーなデータに基づいたリニューアルアウトリーチにより、チャーンを削減し、繰り返し発生する収益を保護します。

### ビジネスへの影響

プロアクティブでパーソナライズされた更新キャンペーンを実装する企業では、通常、更新率が 30～40% 向上し、繰り返し発生する売上高が保護され、顧客の再獲得コストが削減されます。

### 実装方法

[Decisioning とクロスチャネルジャーニー](/help/blueprints/use-case-patterns/campaign-management-orchestration/cross-channel-journey-with-decisioning.md) パターンを使用すると、適切な更新オファーを適切なチャネルで適切なタイミングで提供できます。 このパターンは、journey orchestration と offer decisioning を組み合わせ、アカウント値と使用状況に基づいて動的に更新するインセンティブを有効にします。

### 技術上の考慮事項

- 該当するリードタイム（90、60、30 日など）に、[!DNL Salesforce] または [!DNL Microsoft Dynamics] からトリガーの更新ジャーニーに、契約の終了日、更新条件およびアカウント値をプルします。
- 製品使用状況データと顧客ヘルススコアを組み込んで、更新リスクを判断し、それに応じてオファー戦略をカスタマイズします。
- 決定を使用して、アカウントのライフタイム値とチャーンの確率に基づいて最適な更新インセンティブ（ディスカウント、長期条件、プレミアムアップグレード）を選択します。
- [!DNL Marketo Engage] と CRM タスクの割り当てを通じてメッセージの競合を避けるために、カスタマーサクセスおよびセールスチームとの更新アウトリーチを調整します。


## アップセルと拡張の機会

使用パターン、増加指標、カスタマーサクセスデータに基づいて、製品アップグレードまたは追加ライセンスの準備が整ったお客様を特定します。 積極的な拡大のアウトリーチは、顧客の勢いを売上高の増加に変えます。

### ビジネスへの影響

拡張シグナルを体系的に特定して対応する組織では、通常、拡張収益が 20～30% 増加し、純売上高の保持率と顧客生涯価値が向上します。

### 実装方法

[Decisioning とクロスチャネルジャーニー](/help/blueprints/use-case-patterns/campaign-management-orchestration/cross-channel-journey-with-decisioning.md) パターンを使用し、リアルタイムの使用状況とアカウントのシグナルに基づいて、パーソナライズされたアップセルと拡張オファーを提供します。 このパターンでは、意思決定機能を使用して、チャネル全体で最も関連性の高い拡張オファーと各アカウントを照合します。

### 技術上の考慮事項

- 製品の使用状況テレメトリを取り込んで、ライセンス使用率のしきい値、機能の上限ヒット数、ユーザー数の増加などの拡張信号を検出します。
- 過去の拡張データ、使用状況のトレンドおよびグラフィック属性を使用して、アカウントレベルの傾向モデルを作成します。
- 拡張の機会と推奨オファーを [!DNL Salesforce] 以 [!DNL Microsoft Dynamics] に同期させて、セールスチームがマーケティングで生成されたパイプラインをフォローアップできるようにします。
- [!DNL Marketo Engage] と調整し、拡張メッセージが継続的なサポートやリニューアル通信と競合しないようにします。


## 競合製品の代替キャンペーン

パーソナライズされたメッセージ、移行オファー、競合製品との比較を行うことで、競合製品を使用している見込み客をターゲットに設定します。 各競合他社に関連する特定のペイン・ポイント（問題点）に対処することで、マーケティング・チームはより効果的に差別化を図り、切り替えの意思決定を迅速化できます。

### ビジネスへの影響

ターゲットを絞った競争力のある代替キャンペーンを実施している B2B 組織は、通常、競争力のある獲得率を 15～25% 向上させ、市場シェアを獲得し、現職のベンダーを排除します。

### 実装方法

[ マルチステップオーケストレーションされたジャーニー](/help/blueprints/use-case-patterns/campaign-management-orchestration/multi-step-orchestrated-journey.md) パターンを使用して、特定の競合他社や見込み客のプロファイルに合わせて調整された、マルチタッチの競合キャンペーンを調整します。 このパターンは、特定された競合企業に基づく条件付き分岐をサポートし、各競合シナリオの固有の問題点に対処するメッセージを提供します。

### 技術上の考慮事項

- 競合他社の情報データ（テクノロジー・プロバイダ、CRM の競合他社分野など）を統合プロファイルに統合し、見込み客が現在使用している競合他社を特定します。
- 競合他社固有のコンテンツアセット（移行ガイド、比較シート、ROI 計算ツール）を作成し、それらをジャーニーコンテンツブロックにマッピングします。
- [!DNL Marketo Engage] と連携して、すでにアクティブなセールス・サイクルに入っている見込み客や既存のお客様に対する競合キャンペーンを抑制します。
- 専用のキャンペーンアトリビューションを使用して [!DNL Salesforce] または [!DNL Microsoft Dynamics] で競争力のある変位パイプラインを追跡し、プログラムの有効性を測定します。


## ウェビナーとデモのスケジュール

見込み客の関心、業界、エンゲージメント履歴に基づいて、ウェビナーの招待状やデモスケジュールをパーソナライズします。 適切でタイムリーな招待状は、出席者を増やし、ライブのインタラクションから高品質のパイプラインを生成します。

### ビジネスへの影響

パーソナライズされたウェビナーやデモの招待プログラムにより、通常、ウェビナーの出席率が 35～45% 向上し、ライブエンゲージメントの機会からより適格なパイプラインが促進されます。

### 実装方法

[ イベントトリガーメッセージ ](/help/blueprints/use-case-patterns/campaign-management-orchestration/event-triggered-messaging.md) パターンを使用すると、今後のウェビナーやデモトピックに沿った興味シグナルが見込み客に示された場合に、パーソナライズされた招待状を送信できます。 このパターンは、行動のトリガーにリアルタイムで対応し、興味が最も高いときに招待状が届くようにします。

### 技術上の考慮事項

- 関心に基づくトリガーイベント（製品ページへのアクセス、関連リソースのダウンロード、競合他社比較など）を、予定されているウェビナーやデモトピックに合わせて定義します。
- ウェビナープラットフォームデータ（[!DNL ON24]、[!DNL Zoom Webinar] など）を統合して、統合プロファイル内の登録、出席、エンゲージメントの指標を追跡します。
- デモリクエストとスケジュール設定データを [!DNL Salesforce] または [!DNL Microsoft Dynamics] と同期して、セールスチームがタイムリーな通知とコンテキストを受け取れるようにします。
- 招待の頻度とコミュニケーションの制限 [!DNL Marketo Engage] 調整して、複数のイベントの対象となる過剰なメッセージングの見込み客を防ぎます。


## 導入事例と ROIPersonalization

見込み客の業界、会社の規模、ユースケースに基づいて、パーソナライズされたケーススタディ、ROI 計算、成功事例を提供します。 見込み客が類似した組織のプルーフポイントを見ると、信頼の構築が速くなり、購入の意思決定が加速します。

### ビジネスへの影響

プルーフポイントコンテンツをパーソナライズする組織では、通常、ケーススタディのエンゲージメントが 25～35% 増加し、契約の信頼性と成約率の向上に貢献します。

### 実装方法

[ 既知の訪問者 Web / アプリPersonalization](/help/blueprints/use-case-patterns/personalization/known-visitor-web-app-personalization.md) パターンを使用して、最も関連性の高いケーススタディや ROI 証拠を各見込み客のプロファイルに基づいて動的に提示します。 このパターンでは、訪問者属性がコンテンツライブラリと照合され、すべての見込み客が業界やピアグループからのプルーフポイントを確実に見られるようになります。

### 技術上の考慮事項

- ケーススタディと ROI コンテンツアセットをメタデータ（業界、会社の規模、ユースケース、製品）でタグ付けし、見込み客プロファイルに対する動的なマッチングを可能にします。
- 最初に業界の一致、次に会社の規模とユースケースを優先させるパーソナライゼーションルールを実装し、プロファイルデータが限られている見込み客に対してフォールバックコンテンツを適用します。
- コンテンツエンゲージメントのシグナル（ダウンロード、ページ滞在時間、共有）を統合されたプロファイルに統合し、リードのスコアリングと販売アウトリーチを通知します。
- [!DNL Marketo Engage] と連携して、オンサイトエクスペリエンスだけでなく、メールのシーケンスにパーソナライズされたプルーフポイントコンテンツを含めます。


## 顧客アドボカシープログラム

使用状況と満足度のデータに基づいて、リファレンス、導入事例、証言など、アドボカシーの機会を求めて満足しているお客様を特定して関与させます。 幸せな顧客は、成功を共有するために認識され、招待されたときに強力な成長エンジンです。

### ビジネスへの影響

構造化された顧客アドボカシープログラムは、アドボカシー参加を通常 20～30% 増加させ、新しいビジネスの獲得をサポートする参照、ケーススタディ、証言の絶え間ない流れを生み出します。

### 実装方法

[ マルチステップオーケストレーションジャーニー](/help/blueprints/use-case-patterns/campaign-management-orchestration/multi-step-orchestrated-journey.md) パターンを使用して、満足度と使用状況のシグナルに応えるアドボカシーの特定とエンゲージメントワークフローを構築します。 このパターンは、軽い参加（レビューなど）から始まり、より深いコミットメント（例えば、リファレンスコールやケーススタディ）に進むというプログレッシブアドボカシーの質問をサポートします。

### 技術上の考慮事項

- 満足度サーベイの結果（例：ネットプロモータースコア、顧客満足度スコア）と製品使用状況データを取り込み、各アカウントのアドボカシー対応スコアを作成します。
- 満足度のしきい値、使用マイルストーン、アカウントの保有期間を組み合わせたアドボカシー実施要件条件を定義して、時期尚早のアウトリーチを回避します。
- アドボカシーのステータスとパーティシペーションの履歴を [!DNL Salesforce] または [!DNL Microsoft Dynamics] に同期させて、セールスチームが見込み客調査の際にお客様のアドボカシーを参照できるようにします。
- [!DNL Marketo Engage] と連携し、支援活動のエスカレーションや契約更新の交渉中にアドボカシー・アウトリーチを抑制する。
