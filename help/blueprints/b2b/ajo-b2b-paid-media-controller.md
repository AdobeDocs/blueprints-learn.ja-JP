# AJO B2B - Account Journey Orchestration - Paid Media Controller

>[!TIP]
>このブループリントは、[&#x200B; ユースケースパターン &#x200B;](/help/blueprints/use-case-patterns/b2b/paid-media-orchestration.md)としてB2B Activation &amp; Marketingで利用することもできます。

## 概要

B2Bの有料メディアを大規模に運用しているマーケティング部門では、次のような問題が繰り返し発生します。**アカウントが一度に複数のキャンペーンに参加してしまいます** （ペルソナ、カテゴリ認知、ソリューション主導、追求）これは、メッセージを希薄化し、オーディエンスの疲労を招き、LinkedIn Account Match （アカウントの宛先）全体で手作業によるリスト作業（アップロード、除外、抑制）を余儀なくします。 **ウォーターフォール優先順位付け**&#x200B;と&#x200B;**自動キャンペーン割り当て**&#x200B;がなければ、どのアカウントがどのメッセージを受け取るかを決定する場所は一つではなく、操作は拡張されません。

**有料メディアコントローラー**&#x200B;は、この問題に対処する際に最適なソリューションです。 **Adobe Journey Optimizer B2B edition （AJO B2B）**&#x200B;と&#x200B;**Adobe Experience Platform （AEP）**&#x200B;を一緒に使用します。1つの&#x200B;**アカウントジャーニー**&#x200B;がReal-Time CDPから適格アカウントオーディエンスを読み取り、**スプリットパス （ウォーターフォール）ロジック**&#x200B;を適用して、各アカウントを正確に1つのキャンペーン層に割り当て、**各パスを有料メディア宛先（**&#x200B;例、LinkedIn Matched Audiences **）など）にハンドオフ）にします。**&#x200B;その結果、正確な制御、重複の低減、マルチチャネルのB2B有料メディアオーケストレーションの繰り返し可能なパターンが実現しました。

## ユースケース：マーケターの事例：コントローラーが重要な理由

*MayaはグローバルなB2B ブランドの有料メディアをリードしています。 彼女のチームは、基礎的な認知度、カテゴリーの意図（ジャーニー、データ、コンテンツ）、ソリューション主導のプログラム、ペルソナ施策、必須施策など、数多くのキャンペーンを実施しています。 問題があります。*

**問題：**&#x200B;同じアカウントが複数のキャンペーンに表示されます。 購買意欲の高いジャーニーアカウントも、幅広い認知度を示すリストに含まれています。一方、訴求アカウントには、引き続きペルソナの広告が表示されます。 リストのアップロードと除外は手動で行います。 営業担当者が「必須要素」のリストを更新したり、新しいペルソナキャンペーンを開始したりするたびに、オーディエンスを再書き出しし、スプレッドシートで重複を調整し、LinkedInやその他のプラットフォームに再アップロードします。 スピードが遅く、エラーが発生しやすく、規模が拡大しません。

**必要なもの：**&#x200B;すべての適格なアカウントが1回評価され、明確な優先度ルール（ウォーターフォール）を使用して&#x200B;*最も関連性の高い* キャンペーンに割り当てられ、適切な有料メディアの宛先に自動的に送信される1か所。 手作業によるリストの引き継ぎはありません。 データや戦略が変更されると、システムはリストに触れることなく、アカウントを再評価し、キャンペーン間で移動します。

**Adobeの回答：** **AJO B2BとAEPが連携**&#x200B;して、1つの&#x200B;**有料メディアコントローラー** ジャーニーを実行できます。AEPとReal-Time CDPがデータを保持し、マスターの「適格なアカウント」オーディエンスを1つ保持します。AJO B2Bは、**スプリットパスロジック** （ウォーターフォール）を使用してアカウントジャーニーを実行し、各アカウントを適切な階層に振割りりりり→り分けます。例：→→カテゴリーカテゴリーカテゴリーカテゴリーの認知度のカテゴリーカテゴリーの認知度のカテゴリーソリューションソリューションカテゴリーカテゴリーカテゴリーカテゴリー認知度→カテゴリー認知度&#x200B;**カテゴリーカテゴリーカテゴリーカテゴリー優先度**&#x200B;にをにに割ににに割割割割割割割割割割り割りり割割割りりりり当キャンペーン： 信頼できる唯一の情報源はジャーニーであり、手作業によるリストの書き出しはありません。 これが有料メディアコントローラーのパターンです。AdobeがどのようにB2B有料メディアの正確性と拡張性を高めているのかをご紹介します。

## B2B企業にとって重要な理由：

このパターンを採用している組織は、手作業による抑制と除外ロジックを完全に排除できます（例：ジャーニーで処理される重複の100%）。単一のコントローラーを通じて&#x200B;**数万アカウント**&#x200B;に拡張し、どのアカウントがどのキャンペーンに属するかを&#x200B;**信頼できる唯一の情報源**&#x200B;を維持できます。 リストの再エクスポートや各プラットフォームへの再アップロードを行うことなく、キャンペーンの焦点、ターゲットオーディエンス、販売目標が変更されると、システム **は**&#x200B;を自動的に適応させます。 複数の有料メディアキャンペーンを実施しているB2B企業では、有料メディアコントローラーのパターンにより、手作業でのリストワークフローでは不可能な、明瞭性、コントロール、自動化を実現できます。

次のKPIは、成果測定に適しています。

- **認知度：** ターゲットアカウントは適切な広告を見て、より高い割合で適切なキャンペーンに移動していますか？
- **エンゲージメント：**&#x200B;重複を削除すると、エンゲージメントとコンバージョンが向上しますか？
- **効率：**&#x200B;手動リストの作業（アップロード、除外、抑制）はどの程度減少していますか？
- **コスト：**&#x200B;自動化されたオーケストレーションにより、獲得したアカウントまたは商談あたりのコストはどのように変化しますか？

## 有料メディアジャーニーのオーケストレーション

この設計図の焦点となる一般的なユースケースは、**B2B有料メディアジャーニーオーケストレーション**&#x200B;です。すべての適格なアカウントが、重複や手作業による引き継ぎなしに、正確に1つのキャンペーン層に割り当てられ、適切な有料メディアの宛先にアクティベートされるようにします。

コントローラージャーニー&#x200B;**は、**&#x200B;適格なアカウントオーディエンス （AEP データからReal-Time CDPに組み込まれた）を読み取り、**はスプリットパス （ウォーターフォール）条件（例：ソリューション主導の→ → ペルソナの追求→カテゴリの意図→基礎）を通じて各アカウントを**&#x200B;評価し、**対応する宛先（例：各キャンペーンのLinkedIn Matched Audiences）に各パスをアクティベートします。**

**アカウントに焦点を当てた解決策：**&#x200B;有料メディアコントローラーの焦点は&#x200B;**アカウント**&#x200B;と&#x200B;**キャンペーンの割り当て**&#x200B;です。 技術的な設定では、適格なアカウントと属性（インテント、セグメント、ペルソナなど）を表すデータとオーディエンスがサポートされます。これは、アカウントレベルのセグメンテーションとジャーニーベースのオーケストレーションを成功させるために必要です。

## 要件定義

アカウントに特化したソリューションには、次のアプリケーションとサービスが必要です。

- **Adobe Journey Optimizer B2B edition** — アカウントジャーニー、スプリットパス（ウォーターフォール）ロジック、宛先に対するアクティブ化。
- **Adobe Real-time Customer Data Platform （RTCDP） B2B edition** — アカウントプロファイル、アカウントオーディエンス（有料メディアの適格アカウントなど）。

## アーキテクチャ

高レベルのフロー：

1. **データとオーディエンス** — AEPはプロファイルとイベントを保持します。Real-Time CDP B2Bは、ジャーニー入力オーディエンスとして使用されるアカウントオーディエンス（例：「有料メディアの対象アカウント」）を構築します。
2. **オーケストレーション** — AJO B2B アカウントジャーニー：**オーディエンス** （適格アカウント）→ **パスを分割** （ウォーターフォール：例：ソリューション主導の→の追求→ Persona → Category → Foundational） → **宛先にアクティベート** （LinkedInまたはその他の有料メディアへのパスあたり）。
3. **宛先** – 有料メディアチャネル（LinkedIn Matched Audiencesなど）は、各ジャーニーパスからアカウントレベルのアクティベーションを受け取ります。手動でのリストのアップロードは行われません。

## B2B AEPのデータモデリング

データドリブン型のオーケストレーションでは、スキーマを設計することが重要です。 AEP/RTCDPのアカウントプロファイルと人物プロファイルには、**スプリットパス条件**&#x200B;で使用される属性（例：追跡フラグ、ソリューションの興味、ペルソナ、インテントカテゴリ、エンゲージメントスコア）が含まれている必要があります。 B2B スキーマ（XDM ビジネスアカウント、XDM個人プロファイル、リレーショナル）は、階層とデータソースを表す必要があります。 詳しくは、[RTCDP B2B スキーマ &#x200B;](https://experienceleague.adobe.com/en/docs/experience-platform/rtcdp/b2b-overview)および[AJO B2B ドキュメント &#x200B;](https://experienceleague.adobe.com/en/docs/journey-optimizer-b2b/user/home)を参照してください。

**注：** ジャーニーの分割パスロジックでは、プロファイルと、サポートされている場合はリレーショナルデータが使用されます。ウォーターフォールロジックに必要なフィールドがジャーニーで使用できることを確認してください。

## 実装

次の手順では、AJO B2BおよびAEPを使用した有料メディアコントローラーの実装に関するガイダンスを示します。

### 前提条件のステップ

1. **Real-Time CDP B2Bでアカウントオーディエンスとデータモデルを定義します。**

   コントローラージャーニーに参加する&#x200B;**適格アカウント オーディエンス** （例：「有料メディア対象アカウント」）を作成します。 セグメントビルダーと、適格性を定義するアカウント/個人の属性（地域、セグメント、MQA ステータスなど）を使用します。 このオーディエンスは、ジャーニーの最初の接点です。

2. **キャンペーン階層と分割ロジックを定義します。**

   ウォーターフォールの順序（例：Pursuit → Solution-Led → Persona → Category → Foundational）と各パスの条件（どの属性またはオーディエンスがアカウントに該当するか）を文書化します。 条件が&#x200B;**順序**&#x200B;で相互に排他的であることを確認します（トップダウン）。アカウントは、trueを評価する最初のパスのうち、最大1つのパスに一致する必要があります。

3. **宛先を設定します。**

   AEP/RTCDPで有料メディアの宛先（LinkedIn Matched Audiencesなど）を設定し、AJO B2Bからアクティベートできるようにします。 各宛先のアカウント IDと必要な属性をマッピングします。

### 有料メディアコントローラー設定

1. **AJO B2Bでコントローラージャーニーを作成します。**

   - **オーディエンスの読み取り：** Real-Time CDPから適格なアカウントオーディエンスを選択します。
   - **パスを分割：** パス 1を最優先事項とし、優先順に進むように、各有料メディアオーディエンスのパスを作成します。 パスごとに属性を追加して、クオリフィケーションの基準を設定します（例：「オーディエンスを追求する」、「ソリューションの関心= X」、「ペルソナ = Y」、「インテント カテゴリ = Z」）。 アカウントは、スプリットパスノード aをウォーターフォール形式で評価し、条件を満たす最初のパスの対象となります。
   - **宛先に対してアクティブ化：** パスごとに、「宛先に対してアクティブ化」ノードを正しいLinkedIn （または他の）キャンペーン/宛先に追加します。

2. **相互の排他性を検証します。**

   - 各アカウントが1つのパス（最初に一致する条件）のみを入力することを確認します。
   - アクティベーションを確認：アカウントは適切な宛先に表示され、意図したとおりに優先度の低いキャンペーンから除外されます。

### Audience Activation

1. **LinkedIn （およびその他の宛先）に対してアクティブ化します。**

   ジャーニーの「宛先にアクティベート」を使用して、各パスを適切な有料メディアオーディエンスに送信します。 手作業によるリストの書き出しやアップロードは不要で、アカウントがパスを流れる際にジャーニーがアクティベーションを促進します。

2. **監視と調整**

   ジャーニーレポートを使用して、パスごとのボリュームを監視します。

## 概要

**Paid Media Controller**&#x200B;のブループリントは、**AJO B2BとAEP**&#x200B;が連携して、アカウントをキャンペーンに割り当て、有料メディアにアクティベートするための単一の自動化方法をB2B マーケターに提供する方法を示しています。1つのマスターオーディエンス、1つのジャーニー、ウォーターフォール分割ロジック、宛先への直接活性化で、手作業によるリストの引き継ぎはありません。 これにより、マルチチャネルのB2B有料メディアオーケストレーションのための反復可能なパターンを確立し、重複を減らし、関連性を向上させ、運用を拡大するのに役立ちます。
